# Описание каталога `erp_25/`

Каталог `erp_25/` содержит набор Markdown-документов. Каждый документ описывает оценку доработки/покрытия по отдельному требованию (в заголовках обычно указан формат `Оценка покрытия требования [строка N]`).

## Что внутри `erp_25/`

Документы сгруппированы по сквозным участкам процесса:

- Ведение/подготовка НСИ и категоризация номенклатуры (`10_*` ... `13_*`).
- Формирование и корректировка потребностей, заявочных/плановых кампаний (`14_*` ... `21_*`).
- Планирование закупок, тендеров и обработка предложений поставщиков (`22_*` ... `34_*`).
- Согласования/контроль этапов закупки и поставок (`35_*` ... `41_*`).
- Нормативы и учет СИЗ: выдача, карточки, контроль перемещений (`42_*` ... `45_*`).
- Прослеживаемость и контур РНПТ: коды ТН ВЭД, проверка через ФНС, ЭДО и контроль остатков (`46_*` ... `51_*`).
- Отчеты и аналитика по состоянию потребности/обеспеченности и план-факту (`52_*` ... `59_*`).
- Интеграции с внешними системами: `ERP`, `ИСУ Тендер`, `ИСУ Снабжение`, `КИС КССС`, `КИС КЭД.БД` (`60_*` ... `64_*`).

## Расшифровка сокращений

- `ERP` — Enterprise Resource Planning, планирование ресурсов предприятия.
- `ИСУ` — Информационная система управления (в проекте используется как интеграционный контур для тендера/снабжения).
- `КИС` — Корпоративная информационная система (внешние контуры в рамках интеграций).
- `КЭД` — обозначение внешнего контура/системы (точное значение определяется документацией заказчика; в файлах используется как префикс `КИС КЭД`).
- `КССС` — обозначение внешнего контура/системы (точное значение определяется документацией заказчика; в файлах используется как префикс `КИС КССС`).
- `НСИ` — нормативно-справочная информация.
- `ТРУ` — товары, работы, услуги.
- `СИЗ` — средства индивидуальной защиты.
- `ТН ВЭД` / `ТНВЭД` — Товарная номенклатура внешнеэкономической деятельности.
- `РНПТ` — реестр национальной прослеживаемости товаров.
- `ФНС` — Федеральная налоговая служба.
- `ЭДО` — электронный документооборот.
- `ГТД` — грузовая таможенная декларация.
- `УПД` — универсальный передаточный документ.
- `СФ` — счет-фактура.
- `SLA` — Service Level Agreement, соглашение об уровне сервиса.
- `RLS` — Row-Level Security, разграничение доступа на уровне строк.
- `MCP` / `RLM` — внутренние сокращения в документации оценки (обозначают разделы/категории по статусу подтверждения: «факты/гипотезы/пробелы»).

## Единицы оценки и трудозатраты

Внутри документов трудозатраты приводятся в человеко-днях: `чел.-дн.` (как правило, с диапазонами `Min / Most likely / Max` и суммарным `ИТОГО`).

**Оценка выполнена в человеко-днях (чел.-дн.).**
