# Оценка доработки: Автоматический подбор аналогов в документах

## 1) Карточка оценки

- Requirement ID: строка требования 13
- Название: Автоматический подбор аналогов в документах
- Группа функциональных требований: Ведение НСИ
- Дата оценки: 2026-03-19
- Версия оценки: v1

## 2) Краткая формулировка требования

Обеспечить автоматический подбор аналогов номенклатуры при работе с документами.

## 3) Подтверждение текущей реализации (факты / гипотезы / пробелы)

### Подтвержденные факты (MCP)

- В `ОбщиеМодули.АналогиМатериалов` есть экспортная функция `ВыполнитьЗаменуНаАналоги(...)` с описанием: выполняет замену материалов на подобранные аналоги.
- В `ОбщиеМодули.АналогиМатериаловКлиент.ОткрытьПодборАналогов(...)` открывается форма `Документ.РазрешениеНаЗаменуМатериалов.Форма.ПодборАналогов`.
- В форме документа `Документы.ЗаказПоставщику.ФормаДокумента` есть процедура `ПодобратьАналоги(...)`, которая вызывает `ОткрытьПодборАналогов()`.
- В функции `Документы.ЗаказПоставщику.ФормаДокумента.ПараметрыВыбораАналогов(ИмяТаблицы, Режим)` зафиксированы режимы:
  - `0 - показать аналоги`;
  - `1 - подобрать аналоги`;
  - `2 - заменить аналогами`.
- Аналогичная функция `ПараметрыВыбораАналогов(...)` подтверждена в документах:
  - `Документы.ЗаказМатериаловВПроизводство`,
  - `Документы.ПланЗакупок`,
  - `Документы.ПланПроизводства`,
  - `Документы.ЭтапПроизводства2_2`,
  а также в профильных обработках редактирования этапов/спецификаций.
- В описании метаданных `Документы.РазрешениеНаЗаменуМатериалов` прямо указано применение для ручного подбора в документах и для автоматической замены на аналоги в производственных сценариях.

### Гипотезы/предположения

- Требование трактуется как наличие механизма автоматизированного подбора/замены аналогов в документных сценариях, без обязательного требования к полностью фоновому (без команды пользователя) запуску в каждом типе документа.

### Пробелы (нужны данные для точной оценки)

- Подтвержденных обязательных функциональных пробелов по формулировке требования не выявлено.

## 4) Статус реализации

- Полностью реализовано:
  - механизм подбора и последующей замены аналогами в документах;
  - единая параметризация режимов подбора (`показать/подобрать/заменить`);
  - масштабирование механизма на набор документов закупки/производства.
- Частично реализовано:
  - не требуется.
- Не реализовано (по подтвержденным данным):
  - не выявлено.

## 10) Допущения и валидность оценки

### Допущения

- Под автоматическим подбором понимается подтвержденный в коде режим `подобрать аналоги` с последующей автоматизированной заменой в табличных частях документов.
- Оценка дана для сценария верификации уже реализованного функционала.

### Когда оценка перестанет быть актуальной

- Если будет уточнение, что требуется полностью бездействий пользователя (фоновый автоподбор без команды) для всех целевых документов.
- Если появятся новые обязательные требования по правам, RLS или интеграциям.
