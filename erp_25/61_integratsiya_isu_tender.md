# Оценка: [строка 61] Интеграция с ИСУ Тендер

## 1. Карточка оценки
- Номер строки: 61
- Название: Интеграция с ИСУ Тендер (лоты, победитель, цены)
- Краткая формулировка требования: исходящая передача лотов, входящая загрузка сведений о победителе и ценах победителя.
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: связать контур закупок/лотирования в 1С с внешней тендерной платформой без ручного дублирования данных.
- Границы доработки (scope / out-of-scope): **scope** — обмен по согласованному API/файлу для лотов и результатов тендера, сопоставление ключей, загрузка в объекты цен/результатов; **out-of-scope** — функциональность самой ИСУ Тендер, юридическое проведение процедур закупки вне 1С.

## 2. Статус реализации
- Полностью реализовано: нет (отдельный контур «ИСУ Тендер» в метаданных не выявлен).
- Частично реализовано: **вероятно** — смежные артефакты домена тендера в связке с КИС КЭД и XDTO для лотов (см. факты); прямое соответствие требованию не установлено.
- Не реализовано: не утверждается (возможна реализация под другим именованием или вне индексируемых синонимов).
- Не определено (требует проверки): да — полный сценарий «лот → ИСУ Тендер → победитель → цены» по BSL и точкам вызова.
- Вероятно покрыто частично: перечисление статусов тендера КИС КЭД, пакеты XDTO `zerno_api_lots_*` (назначение относительно ИСУ Тендер требует бизнес-подтверждения).
- Требует проверки: идентификация внешней системы (точное имя, протокол); не путать с интеграцией ЗЕРНО/ЕИС и др.
- Пояснение статуса (1-3 предложения): По семантическому поиску и перечню объектов с наименованием «Тендер» найдены в основном объекты контекста КИС КЭД; отдельной подсистемы «ИСУ Тендер» не обнаружено. Без трассировки кода нельзя подтвердить автоматическую выгрузку лотов и обратную загрузку победителя именно в эту систему.

## 3. Подтвержденные факты
- Подтверждено метаданными: перечисление `лтxСтатусыТендераКИС_КЭД`, справочник `лтхОснованияНепроведенияТендераКИС_КЭД` — признаки учёта статусов тендера в интеграции с КИС КЭД.
- Подтверждено метаданными: пакеты XDTO `zerno_api_lots_1_0_3` и `zerno_api_lots_1_0_6` (по наименованию — формализованный контракт данных по лотам; прямое отнесение к «ИСУ Тендер» метаданными не поясняется).
- Подтверждено метаданными: регламентное задание `ПолучениеЭлектронныхДокументовИзЕИС` (сфера закупок) — смежный, но не эквивалентен формулировке требования.
