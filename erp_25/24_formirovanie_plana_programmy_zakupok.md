## 1. Карточка оценки
- Номер строки: 24
- Название: План закупок — формирование плана/программы закупок
- Краткая формулировка требования: Сводный план/программа закупок с группировкой по категориям и перечням ТРУ; в формулировке зафиксирована доработка относительно типового.
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: Получить управляемый юридически и аналитически значимый документ ПЗ с разрезами для закупочной политики, отчётности и внешних выгрузок.
- Границы доработки (scope / out-of-scope): В scope: структура документа/регистра ПЗ, группировка и представление по перечням ТРУ, связь с заявками/планом потребности, подготовка к интеграции с ИСУ Снабжение (62). Out-of-scope: лотирование (28), тендеры (29–32), заказы (37) — только пересечения по связям.

Явной группировки и идентификации позиций по «перечням ТРУ» в табличной части документа не подтверждено; классификация ТРУ доступна на уровне номенклатуры через реквизит `КодТРУ`. Интеграционный контур ИСУ Снабжение по ПЗ в метаданных отдельно не подтверждён.

## 3. Подтвержденные факты.
- Подтверждено метаданными: существует документ `Документы.ПланЗакупок` с реквизитами периода, сценария, вида плана, статуса (`Перечисление.СтатусыПланов`), складов/партнёров и табличной частью `Товары` (номенклатура, характеристика, количество, назначение, даты и пр.).
- Подтверждено метаданными: у справочника `Номенклатура` есть реквизит `КодТРУ`, пригодный для отчётной группировки по ТРУ при заполненности данных.
- Подтверждено BSL-кодом: в модуле объекта `ПланЗакупок` при записи вызывается `Планирование.ПроверитьСтатусУтвержден` для типа плана закупок; реализованы блокировки и логика, связанная с замещением планов (`Замещающий`, обновление регистра `ЗамещениеПланов`).
- Подтверждено связанным отчётом оценки по строке 22: отдельный документ «заявка на закупку» как промежуточная стадия не найден; переход к закупкам опирается на типовой `ПланЗакупок` и `ЗаказПоставщику`.
