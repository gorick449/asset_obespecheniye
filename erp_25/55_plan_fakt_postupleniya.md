# Оценка: [строка 55] Отчёт «План-факт поступления»

## 1. Карточка оценки
- **Номер строки:** 55
- **Название:** План-факт поступления
- **Дата оценки:** 23.03.2026
- **Версия оценки:** v1
- **Цель:** сравнение ожидаемых дат/количеств с фактом поступления по заказам поставщикам и накладным.
- **Границы:** отчёт по заказам/приобретениям/поступлениям; без изменения договорной архитектуры.

## 2. Статус реализации
- **Полностью реализовано:** нет (как отдельный отчёт с названием «план-факт поступления» не выделен)
- **Частично реализовано:** да — типовой отчёт **ИсполнениеПланаЗакупок** сравнивает план закупок с заказами и фактом; документ **ЗаказПоставщику** содержит статусы и привязку к поступлениям (описание в справке метаданных); отчёт **КонтрольОформленияТоваровВПутиИНеотфактурованныхПоставок** для контуров «в пути».
- **Пояснение:** функциональность план-факта частично покрывается типовыми отчётами исполнения планов и контроля поставок; требуется проверка соответствия формулировке 55 (разрезы поставщик, договор, дата).

## 3. Подтверждённые факты
- Отчёт **ИсполнениеПланаЗакупок** предназначен для контроля выполнения плана закупок с показателями план/факт/невыполненные заказы (справка метаданных).
- Регистр **РасчетыСПоставщикамиПланПоставок** / **ЗаказыПоставщикам** присутствуют в метаданных как основа плановых и заказных данных.

## 4. Пробелы
- Явная связь «строка потребности — заказ — поступление» для отчёта 55 при кастомной заявочной цепочке.

## 5. Новые объекты
- Чаще не нужны — достаточно варианта СКД.

## 6. Существующие объекты

## 7. Зависимости
- 37, 38, 57.

## 8. Роли
- Закупки, склад — чтение; при необходимости маскирование цен.

## 9. Варианты
1. **Минимально инвазивный:** пользовательский вариант **ИсполнениеПланаЗакупок** + отбор по дате поступления.
2. **Архитектурный:** отдельный отчёт «План-факт поступления» с унификацией источников плановой даты (заказ, график, договор).

## 10. Класс сложности
