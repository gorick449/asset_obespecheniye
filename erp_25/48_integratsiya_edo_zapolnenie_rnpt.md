## 1. Карточка оценки
- Номер строки: 48
- Название: Учёт РНПТ — интеграция с ЭДО для заполнения РНПТ из входящих счетов-фактур
- Краткая формулировка требования: При получении электронных первичных документов (в т.ч. УПД/счёт-фактура) сведения о прослеживаемости и РНПТ должны переноситься в учётные документы закупки.
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: Исключить ручной ввод РНПТ при юридически значимом электронном потоке от поставщиков.
- Границы доработки (scope / out-of-scope): В scope: типовой обмен с контрагентами (БЭД), сопоставление номенклатуры, перенос прослеживаемости в строки товаров. Out-of-scope: доработка форматов ФНС на стороне оператора ЭДО.

## 2. Статус реализации
- Полностью реализовано: нет (без подтверждения всех цепочек для нетиповых документов поступления)
- Частично реализовано: да
- Не реализовано: нет
- Не определено (требует проверки): да — покрытие кастомного контура закупок проекта
- Вероятно покрыто частично: нет
- Требует проверки: сценарий «входящий УПД → приобретение/поступление» в рабочей ИБ
- Пояснение статуса (1-3 предложения): Подтверждена развитая подсистема ЭДО (модули и объекты учёта записей ЭДО, форматы ФНС для УПД, подписка на заполнение счёта-фактуры из распознанного документа). Подтверждено BSL-кодом наличие процедур заполнения строк товаров сведениями о прослеживаемости в модуле `ОбменСКонтрагентамиУТ`. Связь «каждый входящий формат → каждый нетиповой документ поступления» не верифицирована.

## 3. Подтвержденные факты.
- Подтверждено метаданными: присутствуют объекты подсистемы обмена с контрагентами (`Обработки.УчетныеЗаписиЭДО`, `РегистрыСведений.УчетныеЗаписиЭДО`, модули `ФорматыЭДО_ФНС`, `СервисНастроекЭДОПовтИсп` и др.), подписка `ЗаполнитьСчетФактуруИзРаспознанногоДокумента`.
- Подтверждено BSL-кодом (описание процедур): в `ОбщиеМодули.ОбменСКонтрагентамиУТ` есть экспортные процедуры `ЗаполнитьСтрокуТоваровСведениямиОПрослеживаемости` и связанные методы для передачи сведений о прослеживаемости при обмене.
