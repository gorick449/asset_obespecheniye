# Оценка: [строка 63] Интеграция с КИС КССС

## 1. Карточка оценки
- Номер строки: 63
- Название: Интеграция с КИС КССС (коды централизованных НСИ)
- Краткая формулировка требования: получение и учёт корпоративных кодов НСИ из КИС КССС для справочников (в т.ч. контрагенты, номенклатура — по фактическому охвату).
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: единые идентификаторы НСИ для отчётности и межсистемных обменов.
- Границы доработки (scope / out-of-scope): **scope** — поддержка загрузки/обновления кодов, контроль обязательности, регламенты; **out-of-scope** — изменение самой КИС КССС.

## 2. Статус реализации
- Полностью реализовано: **не подтверждено** без проверки охвата всех требуемых справочников и сценариев номенклатуры.
- Частично реализовано: **вероятно нет** — объём метаданных указывает на развитую реализацию контура КССС.
- Не реализовано: нет (как минимум каркас и объекты присутствуют).
- Не определено (требует проверки): полный перечень справочников с обязательным кодом КССС в прикладной логике.
- Вероятно покрыто частично: расширение на дополнительные виды НСИ сверх уже реализованных.
- Требует проверки: обязательность кода для номенклатуры vs только контрагенты.
- Пояснение статуса (1-3 предложения): В конфигурации выделена подсистема `лтхКИС_КССС`, общий модуль `лтхМодульКССС`, WS-ссылка `лтхKSSS_INN_KPP`, обработка `лтхПолучениеКонтрагентовКССС`, регистр `лтхЗагрузкаКСССКонтрагентов`, справочники `лтх_КССС`, `лтх_ВидыИнформацииКССС`, регламент `лтхЗаполнениеКодаКСССКонтрагенты`, константа запрета создания контрагента без КССС. Это сильные индикаторы реализованного контура.

## 3. Подтвержденные факты
- Подтверждено метаданными: подсистема `Подсистемы.лтхКИС_КССС` с синонимом «Интеграция с КИС КССС».
- Подтверждено метаданными: общая команда `лтхНастройкаИнтеграцииКССС`, функциональная опция `лтхИнтеграцияСКССС`, константа `лтхИспользуетсяИнтеграцияКССС`, константа `лтхТестоваяШинаКССС`.
- Подтверждено метаданными: регистр сведений `лтхЗагрузкаКСССКонтрагентов`, пакет XDTO `лтхКССС_Контрагентов`, обработка `лтхПолучениеКонтрагентовКССС`, регламентное задание заполнения кода КССС у контрагентов.
- Подтверждено метаданными: роли `лтхДобавлениеИзменениеКССС`, `лтхЧтениеКССС`, `лтхЧтениеВидовИнформацииКССС`.
