## 1. Карточка оценки
- Номер строки: 50
- Название: Учёт РНПТ — автоматическое формирование уведомлений об операциях с прослеживаемыми товарами
- Краткая формулировка требования: Система должна поддерживать подготовку и отправку уведомлений в ФНС по операциям с прослеживаемыми товарами с минимизацией ручного ввода (в идеале — автоматизация).
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: Соответствие требованиям ФНС по прослеживаемости без лишней ручной работы.
- Границы доработки (scope / out-of-scope): В scope: документы уведомлений, проведение, обмен с ФНС через типовой контур 1С-Отчётность/БРО. Out-of-scope: изменение XML-форматов ФНС; не путать с рассылкой пользователям (требование 41).

## 2. Статус реализации
- Полностью реализовано: нет (из-за неоднозначности слова «автоматическое» без подтверждения регламентов)
- Частично реализовано: да
- Не реализовано: нет
- Не определено (требует проверки): наличие и расписание регламентных заданий, полностью ли закрыты все виды операций заказчика без ручного запуска
- Вероятно покрыто частично: да — для типовых видов уведомлений
- Требует проверки: сценарии «распоряжения → уведомление» и степень автозаполнения
- Пояснение статуса (1-3 предложения): Подтверждены документы `УведомлениеОВвозеПрослеживаемыхТоваров`, `УведомлениеОбОстаткахПрослеживаемыхТоваров`, `УведомлениеОПеремещенииПрослеживаемыхТоваров`, обработка/документ `ОтразитьОперацииСПрослеживаемымиТоварами`, модуль `ПрослеживаемостьБРУ` с проведением уведомлений, функция `СведенияОПрисвоенииРНПТ` для квитанций ФНС, регламентированные отчёты по прослеживаемости. Полный «zero-touch» по всем операциям и расписанию без анализа регламентных заданий не фиксируется.

## 3. Подтвержденные факты.
- Подтверждено метаданными (справка объектов): документы уведомлений о ввозе, об остатках и о перемещении прослеживаемых товаров с описанием сценариев оформления и связи с приобретением/остатками.
- Подтверждено BSL-кодом (сигнатуры/описание): `ПрослеживаемостьБРУ.ОбработкаПроведенияУведомление` выполняет проведение документов уведомлений об остатках и о ввозе; `ИнтерфейсыВзаимодействияБРОВызовСервера.СведенияОПрисвоенииРНПТ` предназначена для разбора квитанции о присвоении РНПТ (КНД 1169012).
- Подтверждено метаданными: регламентированные отчёты `РегламентированныйОтчетПрослеживаемыеТоварыВвоз`, `...Остатки`, `...ОтчетОбОперациях`, `...Перемещение`, отчётность `ОтчетностьПоПрослеживаемостиТоваров`.

## 9. Варианты реализации — если релевантно и есть неопределенность.
- Минимально инвазивный: использовать типовые документы уведомлений и рабочие места оформления по распоряжениям; обучить ключевые роли.
- Архитектурный: добавить регламентные задания и/или подписки на проведение документов для автосоздания черновиков уведомлений и постановки в очередь отправки (после отдельного согласования рисков и контроля).

- Условия валидности оценки: нижняя граница соответствует настройке и обучению при принятии типовых сценариев; верхняя — разработка регламентной автоматизации и покрытие нетиповых операций.

- Gate 3: риски автоматизации учтены.
