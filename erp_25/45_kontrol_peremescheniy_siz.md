# Оценка: [строка 45] Контроль перемещения, возврата и списания СИЗ

## 1. Карточка оценки
- Номер строки: 45
- Название: Планирование и учёт СИЗ — контроль перемещения, возврата на склад, списания
- Краткая формулировка требования: замкнутый цикл движения СИЗ с контролем перемещений, возврата и списания; влияние на остатки и отчёт обеспеченности (строка 53).
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: исключить «потерю» выданного СИЗ и обеспечить корректные остатки для планирования.
- Границы доработки (scope / out-of-scope): движение в эксплуатации и на складах ERP; связь с типовыми документами перемещения/списания (строки 38, 39).

## 2. Статус реализации
- Полностью реализовано: нет — отдельных объектов с префиксом `лтх` для «возврата СИЗ», «перемещения СИЗ», «списания СИЗ» в выборке по имени не обнаружено.
- Частично реализовано: да
- Не реализовано: нет (типовые механизмы эксплуатации и склада присутствуют).
- Не определено (требует проверки): насколько полностью регистр `лтхСпецодеждаВыданнаяФизЛицам` корректируется при возврате/списании через типовые документы.
- Вероятно покрыто частично: перемещение и списание из эксплуатации через типовые объекты (в конфигурации есть модули форм `лтхПеремещениеВЭксплуатации`, `лтхСписаниеИзЭксплуатации` — признак доработки типового контура под проект).
- Требует проверки: обязательная связь операций с физлицом и регистром выдачи; отдельный склад СИЗ vs общий склад.
- Пояснение статуса (1-3 предложения): специализированного набора документов только для СИЗ на уровне именования не видно; ожидаемое закрытие требования — через типовой учёт ТМЦ в эксплуатации и складские документы с отборами по группам номенклатуры. Деталь поведения при возврате в части регистра выдачи требует проверки в коде проведения.

## 3. Подтвержденные факты
- Регистр `лтхСпецодеждаВыданнаяФизЛицам` фиксирует выдачу в разрезе физлица и номенклатуры (подтверждено метаданными).
- В конфигурации присутствуют объекты/модули с именами `лтхПеремещениеВЭксплуатации`, `лтхСписаниеИзЭксплуатации` (подтверждено метаданными поиска по имени).
- Типовой документ «Перемещение товаров» и контур внутреннего товародвижения присутствуют в платформе ERP (косвенно — из контекста обработки обеспечения в метаданных).
- Связь с отчётностью по обеспеченности СИЗ (строка 53) зависит от полноты движений (логическая зависимость из `specs/design.md`).
