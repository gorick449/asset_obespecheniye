# Оценка: [строка 54] Отчёт «Состояние запасов»

## 1. Карточка оценки
- **Номер строки:** 54
- **Название:** Состояние запасов (резервы под потребности, невостребованное, аварийные, страховые, ДХНО)
- **Дата оценки:** 23.03.2026
- **Версия оценки:** v1
- **Цель бизнеса:** классификация запасов для управленческих решений и соответствия внутренним нормам.
- **Границы:** отчёт СКД + при необходимости донастройка нормативов (11) и назначений (40); отдельный учёт ДХНО — уточнить использование подсистемы **лтх***.

## 2. Статус реализации
- **Полностью реализовано:** нет
- **Частично реализовано:** да — типовые отчёты по запасам и оборачиваемости (**ОборачиваемостьЗапасов** и др.), кастомные **лтх*** отчёты по ДХНО (**лтхДХНО_РеестрыИРасчеты**, **лтхСверкаОстатковДХНО3_0** и связанные объекты); единого отчёта с полным набором категорий из формулировки 54 по метаданным не подтверждено.
- **Не реализовано:** агрегированная модель «резерв / невостребованное / аварийный / страховой / ДХНО» в одном отчёте без доп. настройки.

## 3. Подтверждённые факты
- В отчётах присутствует **ОборачиваемостьЗапасов** с описанием страхового запаса и уровней запаса (метаданные справки).
- Имеется контур **ДХНО** (отчёты и регистры с префиксом **лтх**, напр. **лтхДХНО_РеестрыИРасчеты**).

## 4. Пробелы
- Подтверждение алгоритма «аварийный запас» vs «страховой» vs «ДХНО» — разные источники.
- Резерв под потребности: требуется связка с **назначениями** и регистрами обеспечения.

## 5. Новые объекты
- Обычно не требуются; возможен РС «Классификация запаса» при архитектурном подходе.

## 6. Существующие объекты
