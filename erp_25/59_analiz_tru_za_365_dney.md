# Оценка покрытия требования [строка 59]: анализ ТРУ за 365 дней

## 1. Краткая формулировка требования

- **Цель бизнеса:** анализировать закупки ТРУ за 365 дней по контрагенту, договору, документу, счету учета и перечню ТРУ.
- **Критерии приемки:** есть отчет с фильтром периода 365 дней и нужными разрезами.
- **Границы доработки:** только аналитический отчет.
- **Вне scope:** детализация процесса потребности и графика поставок.

## 2. Статус реализации

**Статус: не подтверждено / вероятно требует доработки.**

Источники данных по закупочным документам подтверждены, но отдельный отчет по ТРУ за 365 дней с указанными разрезами в анализе не подтвержден.

## 3. Подтвержденные факты

- Подтверждены документы `ЗаказПоставщику` и `ПриобретениеТоваровУслуг`.
- В документах подтверждены контрагент, договор, номенклатура, количество, цена, сумма.
- Подтвержден учетный и закупочный контур, достаточный как источник данных для аналитики.

## 4. Гипотезы / предположения

- Отчет можно построить поверх существующих закупочных документов и движений.
- Для привязки к перечню ТРУ может потребоваться дополнительная категоризация из раннего НСИ-контура.

## 5. Пробелы / что не подтверждено

- Не подтвержден сам отчет.
- Не подтверждена группировка по перечню ТРУ.
- Не подтверждены показатели по счету учета в одной аналитике.

## 6. Новые объекты системы

- Вероятен новый отчет СКД.
- При отсутствии категоризации может потребоваться витрина соответствия номенклатуры перечню ТРУ.

## 7. Существующие объекты, требующие доработки

- `Документы.ЗаказПоставщику`
- `Документы.ПриобретениеТоваровУслуг`
- Контур категорий/перечней ТРУ

## 8. Затрагиваемые объекты и модули

- `Документы.ЗаказПоставщику`
- `Документы.ПриобретениеТоваровУслуг`
- НСИ-контур категорий закупки

## 9. Роли, права, RLS

- Права аналитиков и закупщиков.
- Возможны ограничения по организации и контрагенту.

## 10. Варианты реализации

- **Минимальный:** отчет по документам закупки за 365 дней без дополнительной витрины. Оценка: `2 / 4 / 6`.
- **Расширенный:** витрина ТРУ с аналитикой по перечню и счетам учета. Оценка: `4 / 7 / 11`.

## 11. Риски, открытые вопросы и подтвержденные антипаттерны

- Не определено, что понимается под перечнем ТРУ и где он хранится.
- Не определен источник счета учета для единой аналитики.
- **Подтвержденные антипаттерны в текущей реализации не выявлены.**

## 12. Сложность

- **Класс:** `M`
- **Степень модификации:** средняя

## 13. Оценка трудозатрат

- Аналитика: `0.5 / 1.5 / 2.5`
- Разработка: `1.5 / 3.5 / 5`
- Тестирование: `0.5 / 1 / 2`
- Резерв: `0.5 / 1 / 2`
- **Итого:** `3 / 7 / 11.5`

## 14. Допущения, валидность оценки и что проверить

- Оценка валидна, если данные ТРУ доступны через существующий НСИ-контур.
- Нужно уточнить состав показателей и правила фильтрации периода 365 дней.
- Нужно проверить требуемую производительность на объеме документов.

## 15. Проверка quality gates

- **Полнота требования:** достаточна.
- **Подтверждение реализации:** источники данных подтверждены, отчет — нет.
- **Риски и зависимости:** отражены.
- **Сложность и трудозатраты:** указаны.
- **Валидность оценки:** зависит от готовности классификатора ТРУ.
