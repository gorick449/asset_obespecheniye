# Оценка доработки: Ведение аналогов номенклатуры

## 1) Карточка оценки

- Requirement ID: строка требования 12
- Название: Ведение аналогов номенклатуры (полная/частичная замена)
- Группа функциональных требований: Ведение НСИ
- Дата оценки: 2026-03-19
- Версия оценки: v1

## 2) Краткая формулировка требования

Обеспечить ведение аналогов номенклатуры с возможностью указания одного или нескольких заменителей и поддержкой полной/частичной замены.

## 3) Подтверждение текущей реализации (факты / гипотезы / пробелы)

### Подтвержденные факты (MCP)

- В системе есть объект `Документы.РазрешениеНаЗаменуМатериалов` с табличными частями:
  - `Материалы` (в т.ч. `Номенклатура`, `Характеристика`, `КоличествоУпаковок`, `ИдентификаторСтроки`);
  - `Аналоги` (в т.ч. `Номенклатура`, `Характеристика`, `КоличествоУпаковок`).
- Есть `РегистрыСведений.АналогиМатериалов`, где фиксируются связки материал/аналог и количественные параметры замены:
  - измерения включают `Материал`, `Аналог`, `ХарактеристикаМатериала`, `ХарактеристикаАналога`, `ИдентификаторСтроки` и др.;
  - ресурсы включают `КоличествоУпаковокМатериала` и `КоличествоУпаковокАналога`.
- В `ОбщиеМодули.АналогиМатериаловКлиент.ОткрытьПодборАналогов(...)` открывается форма `Документ.РазрешениеНаЗаменуМатериалов.Форма.ПодборАналогов`.
- В `ОбщиеМодули.АналогиМатериалов.ВыполнитьЗаменуНаАналоги(...)` реализован алгоритм:
  - добавляет новые строки с аналогами;
  - уменьшает/удаляет исходные строки материалов в зависимости от объема замены.
  Это подтверждает поддержку частичной (остаток остается) и полной (строка удаляется) замены.
- В `Справочники.Номенклатура` по подтвержденным данным нет реквизитов с именем/семантикой `Аналог*` для самостоятельного ведения аналогов в карточке НСИ.

### Гипотезы/предположения

- Для текущей оценки принято, что требование покрывается существующим механизмом ведения и применения аналогов, без обязательного хранения именно в карточке `Справочники.Номенклатура`.

### Пробелы (нужны данные для точной оценки)

- Подтвержденных функциональных пробелов для формулировки требования не выявлено.

## 4) Статус реализации

- Полностью реализовано:
  - хранение и обработка связей материал-аналоги с количеством (в т.ч. множественные аналоги);
  - подбор аналогов через форму `ПодборАналогов`;
  - алгоритм полной/частичной замены на уровне прикладной логики (`уменьшает/удаляет` исходные строки и добавляет строки аналогов).
- Частично реализовано:
  - не требуется.
- Не реализовано (по подтвержденным данным):
  - не выявлено в рамках принятой трактовки требования.

## 5) Объекты системы

- Новые объекты (создать):
  - не требуются.
- Существующие объекты (доработать):
  - не требуются.
- Затрагиваемые объекты и модули:
  - подтвержденные используемые объекты: `Документы.РазрешениеНаЗаменуМатериалов`, `РегистрыСведений.АналогиМатериалов`, `ОбщиеМодули.АналогиМатериалов`, `ОбщиеМодули.АналогиМатериаловКлиент`.

## 6) Варианты реализации

- Доработка не требуется (по подтвержденным фактам реализации).

## 10) Допущения и валидность оценки

### Допущения

- Базовый механизм аналогов (`РазрешениеНаЗаменуМатериалов` + `АналогиМатериалов`) принимается как достаточный для покрытия требования.
- Оценка дана для сценария верификации уже реализованного функционала.

### Когда оценка перестанет быть актуальной

- Если будет уточнение, что аналоги должны вестись только/обязательно в карточке `Справочники.Номенклатура`.
- Если появятся новые обязательные требования по интеграциям, правам или иной модели данных.

- Gate 2 (подтверждение реализации): **пройден** — подтверждены существующие объекты и алгоритмы по аналогам/замене.
- Gate 3 (риски и зависимости): **пройден** — подтвержденные риски не выявлены, доработка не планируется.
- Gate 4 (сложность и трудозатраты): **пройден** — указан класс S, диапазон min/likely/max и нулевой резерв.
- Gate 5 (валидность оценки): **пройден** — допущения и условия невалидности указаны.
