# Оценка: [строка 44] Учёт выдачи СИЗ и карточки учёта

## 1. Карточка оценки
- Номер строки: 44
- Название: Учёт выдачи СИЗ, формирование карточек учёта СИЗ
- Краткая формулировка требования: документальное оформление выдачи, печатные карточки, движения по выданному СИЗ для отчётности.
- Дата оценки: 23.03.2025
- Версия оценки: 1.0
- Цель бизнеса: персонифицированный учёт выдачи и правовая/отраслевая отчётность по СИЗ.
- Границы: объекты 1С контура СИЗ, печатные формы, регистры; вне scope — утверждение макета карточки вне организации.

## 2. Статус реализации
- Полностью реализовано: **близко к да** по кастомному контуру
- Частично реализовано: возможны разрывы по полноте печатной формы «карточка» vs утверждённый бланк
- Не реализовано: не выявлено как критичный пробел на уровне объектов
- Не определено: соответствие отчёта `лэс_ЛичнаяКарточка` всем полям утверждаемой карточки
- Пояснение: Подтверждены документ `лэс_РаздаточнаяВедомостьСИЗ` (синоним «Раздаточная ведомость СИЗ»), связь с `ЛЭС_ЗаявкаНаВыдачуСпецодежды`, движения в регистр `лэс_СпецодеждаДанныеДляЛичнойКарточки`, команда печати «Раздаточная ведомость», отчёт `лэс_ЛичнаяКарточка`.

## 3. Подтвержденные факты
- Документ `лэс_РаздаточнаяВедомостьСИЗ` содержит реквизиты организации, подразделения, штатного подразделения, ссылку на заявку, табличную часть «СИЗ».
- В модуле объекта `ОбработкаПроведения` выполняется запись движений в регистр `лэс_СпецодеждаДанныеДляЛичнойКарточки` с измерениями/ресурсами по физлицу, типу спецодежды, датам выдачи и завершения эксплуатации, количеству.
- Печатная форма (менеджер печати): «Раздаточная ведомость».
- Писатели в регистр: `лэс_РаздаточнаяВедомостьСИЗ`, `лэс_ВводОстатковПоСпецодеждеДляЛичнойКарточки` (подтверждено анализом движений).

## 4. Открытые вопросы / пробелы
- Полное соответствие макета карточки нормативным требованиям (отрасль/заказчик).
- Покрытие всех видов СИЗ и характеристик в ТЧ (структура ТЧ в сводке анализа была пустой для вложенных реквизитов — требуется визуальная проверка формы).

## 5. Новые объекты системы
- При несоответствии бланка: доработка макета и/или отчёта `лэс_ЛичнаяКарточка` без новых документов (класс S).

## 6. Существующие объекты, требующие доработки
- `лэс_РаздаточнаяВедомостьСИЗ` (печать, проведение при расширении реквизитов).
- `лэс_ЛичнаяКарточка` (СКД/макет).
- Регистр `лэс_СпецодеждаДанныеДляЛичнойКарточки` при необходимости новых измерений.
