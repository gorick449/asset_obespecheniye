# Оценка: [строка 33] Загрузка цен победителя

## 1. Карточка оценки
- Номер строки: 33
- Название: Перенос цен победителя в учётные регистры цен
- Краткая формулировка: обработка загрузки; целевой регистр цен; связь с результатом тендера и заказом; период действия, валюта, НДС.
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: автоматическое обновление ценовых условий после процедуры.
- Границы: **scope** — загрузка в **цены номенклатуры поставщиков** (типовой контур) + связь с тендером; **out-of-scope** — пересчёт всех исторических документов.

## 2. Статус реализации
- Частично реализовано: подтверждена обработка **ЗагрузкаЦенПоставщикаИзВнешнихФайлов** (синоним «Загрузка цен поставщика из файлов») как типовой механизм загрузки цен поставщика.
- Не определено: **прямая** связь загрузки с объектом «результат тендера / победитель» (строка 30) и с **ИСУ Тендер** (61) без дополнительной доработки.
- Пояснение: типовая загрузка **закрывает** часть требования по НСИ цен; специфика «победитель тендера» **требует** явной привязки и, вероятно, доработки.

## 3. Подтвержденные факты
- Объект метаданных **Обработка.ЗагрузкаЦенПоставщикаИзВнешнихФайлов** существует (подтверждено разбором XML обработки в BSL-окружении).
- В индексе методов присутствуют обработчики, связанные с формой загрузки (например, сценарии **ОК** в контексте **ЗагрузкаЦенПоставщикаИзВнешнихФайлов**).

## 4. Пробелы
- Подтвердить целевой регистр (типовой **ЦеныНоменклатурыПоставщиков** или аналог) через движения/код обработки.
- Проверить поддержку **периода действия, валюты, НДС** в используемом виде цен.

## 5. Новые объекты
- При необходимости: команда «Загрузить из результата тендера», временная таблица сопоставления, журнал ошибок интеграции 61.

## 6. Существующие объекты

## 7. Зависимости

## 8. Ролевая модель

## 9. Варианты

- **Архитектурный:** прямой программный перенос в регистр цен с транзакцией и аудитом, без промежуточного файла.
