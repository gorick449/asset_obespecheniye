# Оценка: [строка 21] Согласование плана потребности

## 1. Карточка оценки
- Номер строки: 21
- Название: Согласование плана потребности
- Краткая формулировка требования: маршрут согласования, статус «утверждён», блокировка последующих шагов до утверждения, оповещения (41).
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: формальное утверждение сводного плана потребности перед закупками.
- Границы: внешнее утверждение (бумага) — не учитывается как «реализовано в 1С».

## 2. Статус реализации
- Полностью реализовано: **только если** план потребности отождествлён с объектом, поддерживающим `СтатусыПланов`
- Частично реализовано: **да** для `ПланЗакупок` как плана со статусами; **не подтверждено** для отдельного плана потребности (п.20)
- Пояснение: у `Документ.ПланЗакупок` реквизит `Статус` типа `Перечисление.СтатусыПланов` (ВПодготовке, НаУтверждении, Утвержден, Отменен). Это подтверждает **типовой** контур статусов плана **закупок**, что может перекрывать п.21 **при согласованной трактовке** с заказчиком. Отдельный маршрут для «плана потребности» не найден.

## 3. Подтверждённые факты
- `Перечисление.СтатусыПланов` — четыре значения, включая НаУтверждении и Утвержден.
- Документ `ПланЗакупок` содержит реквизит `Статус` указанного типа (разбор `Documents/ПланЗакупок.xml`).

## 4. Открытые вопросы / пробелы
- Реализованы ли блокировки формирования заявок на закупку/выгрузок до утверждения **в коде** (модули документов) — не подтверждено в данной сессии.
- Наличие БП именно для плана потребности vs использование статусов документа.

## 5. Новые объекты системы
- При отдельном плане потребности: БП/задачи или расширение статусной модели с подписками.

## 6. Существующие объекты, требующие доработки
- Документ плана (20/ПЗ), `СогласованиеЗакупки` (если задействовано), подсистема оповещений (41).

## 7. Зависимости

## 8. Ролевая модель и доступ

## 9. Варианты реализации
