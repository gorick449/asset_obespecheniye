# Оценка: [строка 47] Автоматическая проверка РНПТ через веб-сервис ФНС

## 1. Карточка оценки
- Номер строки: 47
- Название: Автоматическая проверка РНПТ через веб-сервис ФНС
- Краткая формулировка: вызов сервиса ФНС для валидации регистрационного номера партии, журналирование, обязательность перед проведением (по требованию заказчика).
- Дата оценки: 23.03.2025
- Версия оценки: 1.0
- Цель бизнеса: снизить риск отказов отчётности и ошибок в РНПТ.
- Границы: клиент-серверный код 1С, настройки подключения, регламент; вне scope — инфраструктура сертификатов и каналов.

## 2. Статус реализации
- Полностью реализовано: **не подтверждено**
- Частично реализовано: **вероятно** — типовой контур содержит процедуры контроля количества и данных РНПТ (`УчетПрослеживаемыхТоваровЛокализация`: `ПроверитьЗаполнениеКоличестваПоРНПТ` и др.), но это **не эквивалентно** автоматической онлайн-проверке через веб-сервис ФНС без доп. доказательств.
- Не определено: наличие вызова внешнего API ФНС для РНПТ в других общих модулях/версии БСП под другим именем метода
- Пояснение: По индексу методов не найдены сигнатуры вида проверки регистрационного номера РНПТ через ФНС (в отличие от проверок рег. номера ПФР). Требуется уточнение по релизу платформы/БСП или включённым опциям.

## 3. Подтвержденные факты
- Используется общий модуль `УчетПрослеживаемыхТоваровЛокализация` с функциями учёта и **проверки заполнения** полей, связанных с РНПТ.
- Модуль `ПрослеживаемостьУП` содержит функции получения данных РНПТ для документов.

## 4. Открытые вопросы / пробелы

- Где должен вызываться запрос: форма номера ГТД, поступление, отдельная обработка?
