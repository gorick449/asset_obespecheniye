# Оценка: [строка 56] Ежемесячный план-факт: закупки, тендеры, поставки

## 1. Карточка оценки
- Номер строки: 56
- Название: План-факт по плану закупок, плану тендеров, плану поставок (ежемесячно)
- Краткая формулировка требования: сводный управленческий отчёт с тремя блоками показателей за месяц с сопоставлением плановых и фактических значений.
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: мониторинг исполнения закупочной программы в разрезе времени.
- Границы: данные в 1С; вне scope — планы, существующие только во внешних файлах без загрузки.

## 2. Статус реализации
- Полностью реализовано: не подтверждено
- Частично реализовано: вероятно по отдельным блокам (закупки vs тендеры)
- Пояснение: «план поставок» как отдельная сущность может отсутствовать — потребуется агрегация из заказов/графиков (57) или расширение НСИ.

## 3. Подтверждённые факты
- В требованиях перечислены связанные контуры: план закупок (24), согласования (26), лоты и тендеры (27–29), заказы и поступления (37–38), график (57), корректировки ПЗ (27).
- Это многокомпонентный отчёт с несколькими источниками данных.

## 4. Открытые вопросы
- Единый или три отдельных отчёта с общей формой.
- Каков состав показателей: суммы, количество лотов, количество процедур, соблюдение сроков.
- Версионность плана закупок (27) при сравнении с фактом.

## 5. Новые объекты
- При разрыве: регистр сведений «Плановые показатели по месяцам» для нормализации планов; иначе — только СКД с сложными соединениями.

## 6. Существующие объекты
- Документы и регистры плана закупок, тендерных процедур, заказов, поступлений.

## 7. Зависимости
- Внутренние: 24, 26–29, 37, 38, 57, 27.

## 8. Ролевая модель
- Доступ к суммовым данным и статусам процедур — ограниченный контур закупок.

## 9. Варианты реализации
- **Минимально инвазивный:** три подотчёта/варианта одного отчёта на общей форме без единого регистра планов.
- **Архитектурный:** регламентное заполнение витрины данных (РС или отдельный регистр накопления) для быстрых месячных срезов по всем трём блокам.

## 10. Класс сложности
- Минимально инвазивный: **L** (много источников и согласование метрик)
- Архитектурный: **XL** (витрина + регламент + отчёт)
