# Оценка: [строка 31] Тендерная документация по шаблонам

## 1. Карточка оценки
- Номер строки: 31
- Название: Формирование тендерной документации по шаблонам
- Краткая формулировка требования: макеты шаблонов, заполнение из данных лота/плана закупок, набор типов документов; отличие от ручной подготовки вне 1С.
- Дата оценки: 23.03.2025
- Версия оценки: 1.0
- Цель бизнеса: ускорить подготовку пакета документов процедуры и снизить ошибки.
- Границы доработки (scope / out-of-scope): печатные формы/выгрузки из 1С; юридическая валидация комплекта — out of scope.

## 2. Статус реализации
- Полностью реализовано: нет
- Частично реализовано: минимально
- Не реализовано: библиотека шаблонов тендерной документации, привязанная к лоту, не подтверждена
- Не определено (требует проверки): использование универсальных печатных форм БСП для части комплекта
- Вероятно покрыто частично: печатная форма «План закупок» как источник структурированных данных, но не как тендерный комплект
- Требует проверки: наличие макетов docx/odt вне типового набора
- Пояснение статуса (1-3 предложения): для объекта лота и тендерного контура в конфигурации по поиску не найдено опоры; генерация тендерного комплекта из 1С с высокой вероятностью потребует разработки шаблонов и кода заполнения.

## 3. Подтвержденные факты
- У документа «План закупок» в метаданных указана печатная форма с представлением «План закупок».
- Объекты «Лот»/«Тендер» для привязки шаблонов в номенклатуре документов по поиску не обнаружены (см. [строка 28–29]).

## 4. Открытые вопросы / пробелы (если есть)
- Перечень обязательных файлов комплекта (извещение, ТЗ, проект договора и т.д.).
- Формат шаблонов (mxl, docx через БСП, внешние файлы).

## 5. Новые объекты системы
- Справочник шаблонов, хранилище файлов, обработка формирования комплекта, связь с лотом/позициями ПЗ.

## 6. Существующие объекты, требующие доработки
- Документ «План закупок», общие модули печати, подсистема БСП «Печать».

## 7. Зависимости
- Внутренние: [строка 28], [строка 24].
- Внешние: корпоративные шаблоны Word от юридической службы.

## 8. Ролевая модель и доступ
- Права на формирование и скачивание комплекта; защита конфиденциальных полей.

## 9. Варианты реализации — если релевантно и есть неопределённость
- Минимально инвазивный: 1–2 ключевых шаблона (например, спецификация из ПЗ) без полного комплекта.
- Архитектурный: каталог шаблонов, параметризация, версии, мультиязычность, выгрузка пакетом.

## 10. Класс сложности

### Вариант «Минимальный набор шаблонов»
- Класс: M
- Обоснование: ограниченное число макетов и простых запросов заполнения.
- Степень модификации: средняя

### Вариант «Полный комплект и каталог шаблонов»
- Класс: L
- Обоснование: много шаблонов, согласование версий, сложная параметризация, тестирование вариантов.
- Степень модификации: высокая

## 15. Допущения и ограничения
- Допущение: используется типовой механизм печати/офисных макетов платформы.
- Ограничение: юридический перечень шаблонов не предоставлен.

## 16. Решение по оценке
- Вывод: типовой печатной формы плана закупок недостаточно для требования; ожидается доработка шаблонов и кода.
- Рекомендованный диапазон: 5.5 / 8.0 / 10.5 и 11.0 / 15.0 / 19.0 чел.-дней.
- Условия валидности: утверждённый перечень шаблонов и полей.

## 17. Проверка quality gates
- Gate 1: требование процессное/документное — нужен перечень шаблонов от заказчика.
- Gate 2: подтверждён пробел по тендерным шаблонам при отсутствии объекта лота в 1С.
