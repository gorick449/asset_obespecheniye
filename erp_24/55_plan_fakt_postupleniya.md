# Оценка: [строка 55] Отчёт «План-факт поступления»

## 1. Карточка оценки
- Номер строки: 55
- Название: План-факт поступления
- Краткая формулировка требования: сопоставление ожидаемых дат/объёмов поступления с фактом по документам поступления с разрезами по заказам/договорам.
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: контроль исполнения графика поставок и отклонений.
- Границы: отчётность в 1С; вне scope — данные о сроках только во внешних системах без выгрузки.

## 2. Статус реализации
- Полностью реализовано: не подтверждено
- Частично реализовано: вероятно (типовые отчёты по заказам и поступлениям могут покрывать часть)
- Пояснение: ключевой риск — единый источник «плана» (строка заказа, дата отгрузки, договорная дата).

## 3. Подтверждённые факты
- Связи в чек-листе: 37 (заказы), 38 (поступление), 57 (график).
- Конфигурация УП предполагает наличие заказов поставщикам и поступлений как типовых объектов (общий контекст прикладного решения).

## 4. Открытые вопросы
- Что считается плановой датой: дата поступления в заказе, дата отгрузки, дата по графику?
- Нужен ли разрез по лотам/тендерам.

## 5. Новые объекты
- При разрыве: только отчёт СКД; при несогласованных датах — доработка реквизитов заказа или регистра плановых дат.

## 6. Существующие объекты
- Заказ поставщику, поступление товаров/услуг, связанные регистры.

## 7. Зависимости
- Внутренние: 37, 38, 57.

## 8. Ролевая модель
- Чтение для снабжения и склада; при выводе цен — ограничение по ролям.

## 9. Варианты реализации
- **Минимально инвазивный:** СКД по заказам и поступлениям с соединением по номенклатуре/заказу.
- **Архитектурный:** унифицированный регистр «Плановые события поставки» для согласования нескольких источников плановых дат.

## 10. Класс сложности

- Архитектурный: **L**
