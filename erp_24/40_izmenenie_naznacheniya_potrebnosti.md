# Оценка: [строка 40] Изменение назначения потребности

## 1. Карточка оценки
- Номер строки: 40
- Название: Переназначение потребности и высвобождение резерва
- Краткая формулировка требования: операции переноса обеспечения с одной потребности на другую и освобождения зарезервированных количеств с корректным отражением в регистрах и отчётах.
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: гибкое перераспределение запасов между заказами/заявками без ручных обходных проводок.
- Границы: учёт обеспечения в 1С; вне scope — изменение договорных обязательств вне системы.

## 2. Статус реализации
- Полностью реализовано: не подтверждено (тип УП содержит механизмы назначений и обеспечения — степень покрытия сценария «перенос/высвобождение» требует проверки)
- Частично реализовано: вероятно
- Пояснение: без deep-dive по цепочке обеспечения и резервирования статус остаётся «требует проверки»; оценка дана на случай кастомизации поверх типового контура.

## 3. Подтверждённые факты
- В требованиях явно указаны связи с документами потребности (15), планом (20), поступлением (38) и отчётами 52, 54.
- Конфигурация относится к классу УП/ERP 2 с подсистемой обеспечения потребностей (подтверждено анализом метаданных по именам подсистем).

## 4. Открытые вопросы
- Используется ли типовой документ/обработка «корректировка назначения» или требуется новый.
- Нужна ли связь с ID потребности (25) при переносе.
- Ограничения по статусам (согласовано/в закупке/получено).

## 5. Новые объекты
- При разрыве: документ «Изменение назначения потребности» с табличной частью переносов; возможно регистр операций для аудита.

## 6. Существующие объекты
- Регистры обеспечения, резервирования, заказы; документы заявок и заказов поставщику (37).

## 7. Зависимости
- Внутренние: 15, 20, 38, 52, 54; возможно 25.

## 8. Ролевая модель
- Операции высокого риска — роли снабжения/логистики с разделением «инициатор/утверждающий» при необходимости.

## 9. Варианты реализации
- **Минимально инвазивный:** использовать типовой механизм смены назначения (если подтверждён в базе) с донастройкой прав и отчётов.
- **Архитектурный:** отдельный регламентированный документ с блокировками по стадиям, протоколом и связью с ID потребности.

## 10. Класс сложности
- Минимально инвазивный: **M** (при наличии типовой основы)
- Архитектурный: **L** (при полном кастомном контуре и множестве проверок)
