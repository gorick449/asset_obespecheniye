# Оценка: [строка 42] Нормативы выдачи СИЗ (отрасль, корпоратив, антропометрия, характеристики)

## 1. Карточка оценки
- Номер строки: 42
- Название: Нормативы выдачи СИЗ (отрасль, корпоратив, антропометрия, характеристики)
- Краткая формулировка требования: структура норм выдачи по должности/виду СИЗ/размерам, связь с характеристиками номенклатуры; отличие от общих нормативов запасов (строка 11).
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: автоматизировать нормирование выдачи СИЗ для охраны труда и планирования (связь со строками 10, 43, 44).
- Границы доработки (scope / out-of-scope):  
  **Scope:** справочники/регистры норм, разрезы отраслевой и корпоративной нормы, антропометрия, связь с характеристиками номенклатуры, отчёты контроля.  
  **Out-of-scope:** юридическая экспертиза норм ОТ вне системы; полная замена учёта выдачи (строка 44) — только стыковка.

## 2. Статус реализации
- Полностью реализовано: не подтверждено для совокупности разрезов «отрасль + корпоратив + антропометрия + характеристики».
- Частично реализовано: вероятно (подсистема «Спецодежда/СИЗ» и кастомная подсистема `лэс_СпецодеждаНСИ` присутствуют в индексе метаданных/семантического поиска).
- Не реализовано: не определено без детального разбора объектов норм.
- Не определено (требует проверки): полная структура норм и покрытие антропометрии в объектах.
- Вероятно покрыто частично: да.
- Требует проверки: соответствие номенклатурным характеристикам (строка 10) в формах подбора/планирования (строка 43).
- Пояснение статуса (1-3 предложения): семантический поиск и подсистемы указывают на наличие функционала СИЗ и кастомного НСИ спецодежды. Явная декомпозиция норм по всем разрезам требования и тела ключевых процедур в рамках данной оценки не зафиксированы как полностью проверенные.

## 3. Подтверждённые факты
- В метаданных/семантическом индексе присутствует подсистема «Спецодежда/СИЗ» и подсистема `лэс_СпецодеждаНСИ` (подтверждено семантическим поиском по описанию конфигурации).
- Обширный кастомный контур с префиксом `лэс_*` присутствует в BSL-индексе (контекст доработок учёта/НСИ — без детализации конкретной структуры норм в данном отчёте).

- При gap: регистр сведений норм выдачи с измерениями (должность, вид СИЗ, размер, тип нормы), справочник антропометрии сотрудника, связь с характеристиками номенклатуры.

## 6. Существующие объекты, требующие доработки
- Объекты подсистемы Спецодежда/СИЗ и кастомные `лэс_*` справочники/документы; отчёты плана СИЗ (строка 43).

## 7. Зависимости
- Внутренние: номенклатура и характеристики (строка 10), план потребности СИЗ (строка 43), учёт выдачи (строка 44).
- Внешние: при необходимости — кадровая система для антропометрии.

## 8. Ролевая модель и доступ
- Роли службы ОТ, кадров, склада СИЗ; ограничение видимости персональных антропометрических данных.

## 9. Варианты реализации
- **Минимально инвазивный:** использовать типовую/существующую модель СИЗ в конфигурации, доработать только недостающие разрезы (например, явное разделение отраслевой и корпоративной нормы) и связь с характеристиками.
- **Архитектурный:** ввести единый регистр норм с версионированием, матрицей соответствий номенклатуре и отдельным контуром антропометрии сотрудников, плюс отчёты контроля несоответствий.

- Обоснование: предметная область богата измерениями; регрессия по плану и выдаче.

### Вариант «Архитектурный»
