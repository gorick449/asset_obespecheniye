# Оценка: [строка 32] Сбор и анализ предложений поставщиков

## 1. Карточка оценки
- Номер строки: 32
- Название: Множественные предложения на лот и сравнительный анализ
- Краткая формулировка: хранение нескольких предложений; сравнение в системе; связь с выбором победителя (30) и анализом цен (58).
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: обоснованный выбор поставщика до фиксации победителя.
- Границы: **scope** — данные предложений по закупочной процедуре/лоту; **out-of-scope** — ЭТП вне 1С без импорта.

## 2. Статус реализации
- Не реализовано для поставщиков на лот: найден документ **КоммерческоеПредложениеКлиенту** (продажи), **не** эквивалентен предложениям поставщиков по лоту.
- Частично: типовые **цены поставщиков** и отчёты по ценам могут использоваться **вручную** без модели «несколько предложений на лот».
- Пояснение: без объекта **лота** (28) модель предложений **не привязана** к процедуре.

## 3. Подтвержденные факты
- В конфигурации присутствует документ **КоммерческоеПредложениеКлиенту** (не целевой для требования 32).
- Подсистема **Цены и скидки** / загрузки цен поставщика существует на уровне конфигурации (косвенно — обработка **ЗагрузкаЦенПоставщикаИзВнешнихФайлов** упоминается в индексе методов).

## 4. Пробелы
- Нужен ли учёт предложений как документов или как табличной части лота.
- Требования к отчёту **Сравнительный анализ цен** (58) — отдельная оценка.

## 5. Новые объекты
- ТЧ **Предложения** у лота или документ **ПредложениеПоставщикаПоЛоту**, регистр для истории, отчёт сравнения.

## 6. Существующие объекты
- Будущий лот (28), цены поставщиков, отчёт 58.

## 7. Зависимости
- **28, 30, 58**

## 8. Ролевая модель
- Закупки — ввод; подразделения — ограниченное чтение.

## 9. Варианты

- **Архитектурный:** отдельные документы предложений, версии, вложения, интеграция с ИСУ.
