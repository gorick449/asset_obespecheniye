# Оценка: [строка 11] Нормативы запасов

## 1. Карточка оценки
- Номер строки: 11
- Название: Нормативы запасов (аварийный, страховой, min-max)
- Краткая формулировка требования: ведение целевых уровней запасов с разделением видов (аварийный, страховой, min/max) в разрезе НСИ/склада и использование в отчёте по состоянию запасов (54).
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: управляемая классификация запасов для контроля ДХНО, резервов и реагирования на сбои поставок.
- Границы доработки (scope / out-of-scope): в пределах конфигурации 1С — объекты установки нормативов, регистры, отчётность; вне scope — методика расчёта нормативов на стороне заказчика без фиксации в системе.

## 2. Статус реализации
- Полностью реализовано: нет (не подтверждено)
- Частично реализовано: вероятно (типовой контур «Запасы» / «Планирование запасов» в УП)
- Не реализовано: не подтверждено
- Не определено (требует проверки): да — детальная структура регистров и раздельность «аварийный/страховой» не подтверждены из-за ограничений выдачи структуры метаданных по ряду объектов в индексе.
- Вероятно покрыто частично: типовое планирование/анализ запасов без явной таксономии требования.
- Требует проверки: связь строк нормативов с категориями отчёта 54; участие в отчёте «Состояние запасов».
- Пояснение статуса (1-3 предложения): по семантическому поиску и именам объектов присутствуют подсистемы и объекты, связанные с запасами и планированием; отдельный контур «нормативы аварийный/страховой» в явном виде не идентифицирован. Нужна ручная проверка в живой базе и сопоставление с регистрами резервов/обеспечения.

## 3. Подтверждённые факты
- Конфигурация проекта: «УправлениеПредприятием» (контур УП/ERP 2).

- Запрос структуры объекта «ПланированиеЗапасов» через индекс вернул пустой набор реквизитов (ограничение индексации, не доказательство отсутствия функциональности).
