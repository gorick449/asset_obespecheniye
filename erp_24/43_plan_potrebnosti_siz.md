# Оценка: [строка 43] Планирование потребности в СИЗ

## 1. Карточка оценки
- Номер строки: 43
- Название: Планирование потребности в СИЗ (статистика, нормы, резервы)
- Краткая формулировка требования: план закупки СИЗ в разрезе номенклатуры и характеристик по статистике потребления, по нормам, с учётом резервов; связь с общим планом закупок при наличии.
- Дата оценки: 23.03.2025
- Версия оценки: 1.0
- Цель бизнеса: согласованный план закупки СИЗ с учётом фактического потребления, нормативов и страховых/производственных резервов.
- Границы доработки (scope / out-of-scope): внутри 1С — объекты планирования СИЗ, регистры/отчёты статистики, увязка с ПЗ (24) и нормами (42), резервы (11). Вне scope — сбор статистики вне учётных документов, внешние регламенты охраны труда без цифрового следа.

## 2. Статус реализации
- Полностью реализовано: нет
- Частично реализовано: да (отдельные элементы контура СИЗ и общее планирование «лэс»)
- Не реализовано: единый документ/обработка плана СИЗ с тремя явными источниками (статистика + нормы + резервы) в одном сценарии
- Не определено (требует проверки): полнота использования документов «Ввод данных планирования ПЭ01/ПЭ02/ПЭ04» для именно закупки СИЗ; наличие отдельных регистров статистики выдачи для планирования
- Вероятно покрыто частично: заполнение заявки по нормам в документе заявки на выдачу спецодежды
- Требует проверки: связь рассчитанного плана СИЗ с программой/планом закупок (24) и перечнями ТРУ
- Пояснение статуса (1-3 предложения): Подтверждены кастомные документы контура выдачи СИЗ и процедуры заполнения по нормам; отдельного объекта «план потребности/закупки СИЗ» с тремя источниками в метаданных не выявлено. Документы планирования ПЭ01–ПЭ04 имеют общую шапку планирования «лэс» без специализации на СИЗ в доступных сведениях.

## 3. Подтвержденные факты
- В конфигурации «Управление предприятием» (версия 2.4.6.174) есть подсистема «Спецодежда/СИЗ».
- Документ `ЛЭС_ЗаявкаНаВыдачуСпецодежды` содержит логику, связанную с нормами (`ЗаполнитьПоНормам`, `ЗаполнитьПоНормамНаСервере` и др.), то есть нормативный источник для потребности в СИЗ используется в сценарии заявки.
- Существуют документы `лэс_ВводДанныхПланированияПЭ01`, `ПЭ02`, `ПЭ04` с общей моделью шапки (головное подразделение, версия сценария планирования и т.д.) — принадлежат контуру планирования «лэс», но их предметная привязка к СИЗ по одним только синонимам не установлена.
- Связанное требование [строка 11] (резервы) и [строка 42] (нормы СИЗ) влияют на полноту оценки; без них нельзя зафиксировать закрытие требования 43.

## 5. Новые объекты системы
- При варианте «архитектурный»: документ или обработка «План потребности/закупки СИЗ»; при необходимости — регистр сведений плановых потребностей/остатков к закупке; схема сопоставления с номенклатурой и характеристиками.
- При варианте «минимально инвазивный»: расширение существующей заявки/обработки расчёта без нового документа (если примется как достаточное по процессу).

## 6. Существующие объекты, требующие доработки
- `ЛЭС_ЗаявкаНаВыдачуСпецодежды` — расширение до планирования закупки или остаётся только выдачей.
- Документы `лэс_ВводДанныхПланированияПЭ*` — уточнение назначения; возможное переиспользование.
- Контур плана закупок (требование 24) — точки приёма рассчитанных потребностей СИЗ.

- Рекомендованный диапазон: минимально инвазивный — **4.5 / 7.0 / 9.5** чел.-дней; архитектурный — **8.0 / 11.0 / 14.0** чел.-дней.
- Условия валидности оценки: подтверждение назначения ПЭ01–ПЭ04; подтверждение источника статистики; согласование с владельцем процесса ПЗ.
