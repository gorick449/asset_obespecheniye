# Оценка: [строка 28] Формирование лотов

## 1. Карточка оценки
- Номер строки: 28
- Название: Формирование лотов
- Краткая формулировка требования: объект лота, источник строк из плана закупок, правила объединения/разбиения, подготовка к выгрузке в ИСУ Тендер.
- Дата оценки: 23.03.2025
- Версия оценки: 1.0
- Цель бизнеса: структурирование позиций для проведения закупочных процедур.
- Границы доработки (scope / out-of-scope): функциональность 1С и обмен; проведение самой процедуры во внешней системе — out of scope разработки 1С.

## 2. Статус реализации
- Полностью реализовано: нет
- Частично реализовано: не подтверждено на уровне метаданных
- Не реализовано: документ/справочник с именем, содержащим «Лот», в категории документов по поиску — не найден
- Не определено (требует проверки): кастомные объекты с неочевидными именами; полная реализация лотов только в ИСУ Тендер
- Вероятно покрыто частично: только через внешнюю систему при наличии интеграции (модули с «ИСУ» в индексе кода присутствуют, детальная связка с лотами не детализировалась)
- Требует проверки: спецификация обмена по стр. 61
- Пояснение статуса (1-3 предложения): в проиндексированной конфигурации не обнаружены прикладные объекты «лот» по результатам поиска документов по шаблону имени; требование, вероятно, требует значимой доработки или опоры на ИСУ Тендер.

## 3. Подтвержденные факты
- Поиск в категории «Документы» по шаблону «Лот» не вернул объектов.
- Документ «План закупок» как источник номенклатурных позиций для закупочного планирования в конфигурации присутствует.
- В кодовой базе присходят модули интеграции с ИСУ (по индексу методов), что создаёт предпосылку для внешнего лотирования.

## 4. Открытые вопросы / пробелы (если есть)
- Является ли ИСУ Тендер системой-мастером для лотов (стр. 61).
- Требуется ли хранение структуры лота в 1С для учёта и отчётности.

## 5. Новые объекты системы
- При полном контуре в 1С: справочник/документ «Лот», ТЧ позиций, связи с «Планом закупок», обработка формирования, регистр статусов лота.

## 6. Существующие объекты, требующие доработки
- Документ «План закупок», обработки выгрузки, общие модули интеграции.

## 7. Зависимости

- Внешние: ИСУ Тендер [строка 61].

## 8. Ролевая модель и доступ

## 9. Варианты реализации — если релевантно и есть неопределённость
- Минимально инвазивный: лоты в ИСУ Тендер; в 1С — обмен наборами строк ПЗ с ключами сопоставления и минимальным учётом статуса.
- Архитектурный: полноценный объект «Лот» в 1С с правилами агрегации и последующей синхронизацией.

## 10. Класс сложности

- Класс: M
- Обоснование: доработка выгрузки/загрузки и сопоставления без полной предметной модели лота в 1С.
- Степень модификации: средняя

### Вариант «Полный контур в 1С»
- Класс: XL

- Степень модификации: высокая
