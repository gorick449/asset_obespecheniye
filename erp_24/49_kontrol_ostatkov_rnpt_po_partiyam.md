# Оценка: [строка 49] Контроль остатков по РНПТ в разрезе партий

## 1. Карточка оценки
- Номер строки: 49
- Название: Контроль остатков по РНПТ в разрезе партий
- Краткая формулировка требования: учёт количества по регистрационным номерам партий, согласованность с количественными запасами, достаточность для уведомлений (50).
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: управленческий и регламентированный контроль прослеживаемости.
- Границы: настройка сертификатов и каналов ФНС — в смежных требованиях 47/50.

## 2. Статус реализации
- Полностью реализовано: **не подтверждено** в полном объёме формулировки «всех партий и всех сценариев» без продуктивной проверки
- Частично реализовано: **да** — типовой контур прослеживаемости УП присутствует
- Пояснение: подтверждены подсистема/объекты прослеживаемости, регистры `ОперацииСПрослеживаемымиТоварами*`, накопления `ПрослеживаемыеТовары*`, набор отчётов с шаблоном имени `*Прослеж*`. Это соответствует **базовой** функциональности контроля остатков прослеживаемых товаров в типовой конфигурации. Соответствие **каждому** сценарию «партия + РНПТ + запасы» и полнота для уведомлений (50) требует точечной верификации.

## 3. Подтверждённые факты
- Подсистема «Прослеживаемость товаров» найдена семантическим поиском по описанию.
- Регистры сведений: `ОперацииСПрослеживаемымиТоварами`, `ОперацииСПрослеживаемымиТоварамиРасширенный`.
- Регистры накопления: `ПрослеживаемыеТоварыВСоставеОС`, `ПрослеживаемыеТоварыОтгруженныеВЕАЭС`.
- Отчёты: среди прочих — `ОстаткиПрослеживаемыхТоваровВСоставеОСНепринятыхКУчету`, регламентированные отчёты по прослеживаемости (поиск по типу «Отчёты»).
- В `Справочник.Номенклатура` подтверждены реквизиты `ПрослеживаемыйТовар`, `КодТНВЭД` (разбор `Catalogs/Номенклатура.xml`).
