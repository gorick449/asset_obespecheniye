# Оценка: [строка 62] Интеграция с ИСУ Снабжение

## 1. Карточка оценки
- Номер строки: 62
- Название: Интеграция с ИСУ Снабжение
- Краткая формулировка требования: выгрузка планов потребности и планов/программы закупки с разрезами по категориям и перечням ТРУ.
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: передать утверждённые планы в корпоративную систему снабжения без ручной консолидации.
- Границы доработки (scope / out-of-scope):
  - **Scope:** формирование выгрузки из документов/регистров планов потребности и ПЗ, фильтры по перечням ТРУ, регламент, протокол, права.
  - **Out-of-scope:** логика ИСУ Снабжение после приёма файла/сообщения; изменение методологии планирования в 1С.

## 2. Статус реализации
- Полностью реализовано: нет.
- Частично реализовано: **не подтверждено** для **ИСУ Снабжение** как целевой системы; в коде присутствуют выгрузки в **ИСУ СИП** из отдельного документа отчётности — иной контекст и, вероятно, иной набор данных.
- Не реализовано: **да** для явного контура «ИСУ Снабжение ↔ планы потребности/ПЗ по перечням ТРУ» по результатам поиска.
- Не определено: полный состав полей, требуемый ИСУ Снабжение; момент выгрузки (после утверждения 21/26).
- Вероятно покрыто частично: низкая без спецификации; возможны кастомные обработки под другим именем.
- Требует проверки: фактические регламенты на площадке заказчика.

**Пояснение статуса:** подтверждённые выгрузки в ИСУ с другим названием **не** закрывают требование 62 без доказательства совпадения целевой системы и состава данных.

## 3. Подтверждённые факты
- Подтверждено BSL-кодом: наличие серверных процедур выгрузки в **ИСУ СИП** в модуле формы документа `ЕАЕ_SIP_отчеты` (множественные команды вида «Выгрузить…ВИСУНаСервере» с описанием «Выгрузка в ИСУ СИП»).
- Подтверждено анализом метаданных: поиск объектов с «Снабж» в имени в доступной выборке **не дал** результата.
- Подтверждено типовым контуром: документ **План закупок** и общие модули планирования существуют как источник данных для потенциальной выгрузки (без привязки к ИСУ Снабжение).

## 4. Открытые вопросы / пробелы
- Требуется ли выгрузка из **плана потребности** как отдельного документа или достаточно ПЗ и заявок на закупку.
- Обязательность разрезов перечня ТРУ (строка 10) в каждой строке выгрузки.
- Формат: файл vs API.

## 5. Новые объекты системы
- Обработка выгрузки, схема XDTO или макет файла, регистр журналирования/статусов выгрузки, возможно настройки отбора по организации/периоду.

## 6. Существующие объекты, требующие доработки
- Документы плана потребности, плана закупок, связанные регистры планирования — команды и отборы для выгрузки утверждённых версий.
- Справочник номенклатуры (перечни ТРУ) — обеспечение заполненности для валидной выгрузки.

## 7. Зависимости
- Внутренние: 10, 20–24, 26 (утверждение), 25 (ID потребности при необходимости трассировки).
- Внешние: ИСУ Снабжение, контракт данных.

## 8. Ролевая модель и доступ
- Роль на запуск выгрузки и просмотр протоколов; исключение выгрузки черновиков без статусов утверждения (по политике).

## 9. Варианты реализации
- **Вариант A:** периодическая **файловая выгрузка** утверждённых планов с параметрами отбора.
- **Вариант B:** **API-интеграция** с инкрементальной отправкой изменений и подтверждением приёма.

## 10. Класс сложности
- **Вариант A:** **L** — несколько источников, разрезы ТРУ, валидации.
- **Вариант B:** **XL** — устойчивость, идемпотентность, обработка ошибок приёма.

- Вероятность доработок: **высокая**.
- Что проверить: у внедрения фактические обмены и названия целевых систем.

## 15. Допущения и ограничения
- Утверждённые статусы планов определены и доступны в данных для отбора.
- Объём строк умеренный на пилоте; иначе увеличить тестирование и резерв.

## 16. Решение по оценке
- Вывод: требование **не реализовано** как интеграция с **ИСУ Снабжение**; есть иной подтверждённый контур к **ИСУ СИП**, не являющийся доказательством покрытия 62.
- Рекомендованный диапазон:
  - Вариант A: **9.5 / 17.5 / 26.0** чел.-дней
  - Вариант B: **17.0 / 29.0 / 41.0** чел.-дней
- Условия валидности: утверждённый формат и эталонные файлы/запросы от ИСУ Снабжение.

## 17. Проверка quality gates
- Gate 1: критерии приёмки по полям и разрезам ТРУ.
- Gate 2: отсутствие прямого подтверждения для ИСУ Снабжение — отражено.
- Gate 3: внешняя зависимость — высокая.
- Gate 4: оценка по вариантам, шаг 0.5.
- Gate 5: при изменении источников (только ПЗ vs план потребности) — пересмотр.
