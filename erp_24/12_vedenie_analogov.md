# Оценка: [строка 12] Ведение аналогов номенклатуры

## 1. Карточка оценки
- Номер строки: 12
- Название: Ведение аналогов (несколько заменителей, полная/частичная замена)
- Краткая формулировка требования: хранение связей «номенклатура — один или несколько аналогов» с признаком полной/частичной замены для последующего подбора и отчётности.
- Дата оценки: 23.03.2026
- Версия оценки: 1.0
- Цель бизнеса: снижение риска дефицита за счёт формализованных замен.
- Границы: внутри 1С — НСИ и регистры связей; вне scope — методика допуска замен вне системы.

## 2. Статус реализации
- Полностью реализовано: не подтверждено
- Частично реализовано: вероятно (в типовом контуре производственного/складского замещения есть процедуры работы с аналогами)
- Не определено: да — нужна проверка наличия отдельного объекта «аналоги» с видом замены и множественностью
- Пояснение: по анализу кода в конфигурации встречаются процедуры подбора/замены на аналог в контексте управления производством; достаточность для требования «несколько заменителей + полная/частичная» в НСИ — отдельная проверка.

## 3. Подтверждённые факты
- В кодовой базе конфигурации присутствуют элементы, связанные с аналогами в производственном сценарии (подтверждено анализом BSL: операции вида замены на аналог, открытие подбора аналогов).
- Требование явно предполагает связь с автоподбором (13) и отчётом по аналогам (51).

## 4. Открытые вопросы / пробелы
- Где хранятся связи: отдельный регистр, табличная часть, справочник пар?
- Есть ли срок действия связи и приоритет при нескольких аналогах?
- Как кодируется «частичная замена» (коэффициент, единица измерения)?

## 5. Новые объекты системы
- При разрыве: регистр сведений «Аналоги номенклатуры» (номенклатура, аналог, вид замены, приоритет, период) или расширение справочника.

## 6. Существующие объекты
- Справочник номенклатуры; формы элементов; возможные документы списания/производства; отчёт 51.

## 7. Зависимости
- Внутренние: 13, 51; при пересечении с производством — модули УправлениеПроизводством.

## 8. Ролевая модель
- Изменение аналогов — ограниченная роль (НСИ/технолог); чтение — широкий круг пользователей снабжения и склада.

## 9. Варианты реализации
- **Минимально инвазивный:** регистр сведений с периодом и видом замены, подключение к форме номенклатуры.
- **Архитектурный:** справочник «Матрица замен» с версиями, согласованием ввода, контролем дублей и правилами приоритета.

## 10. Класс сложности

- Архитектурный: **L** (согласование, контроль целостности, интеграция с несколькими контурами).
