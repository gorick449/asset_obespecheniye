# Оценка покрытия: [строка 23] Согласование заявок на закупку (службы)

## 1. Карточка оценки
- Номер строки: 23
- Название: Согласование заявок на закупку (ГМ, ГЭ, ГТ, метрологи, транспорт, ИТ, АХО)
- Краткая формулировка требования: многоэтапный маршрут с ролями/заместителями, условные ветви по категориям ТРУ, отражение в мониторинге ([строка 52]).
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: межфункциональная экспертиза до лотирования.
- Границы: бизнес-процессы заявок на закупку в УХ 3.3; без детальной настройки конкретной оргструктуры заказчика.

## 2. Статус реализации
- Полностью реализовано: нет
- Частично реализовано: да
- Пояснение: в конфигурации присутствуют бизнес-процессы **Экспертиза заявки на закупку** и **Корректировка заявки на закупку** (двухшаговая схема исполнение/проверка по справке **Корректировка заявки на закупку**). Модуль **Бизнес процессы заявок на закупку** содержит процедуры первоначального заполнения параметров согласования. Прямое соответствие ролям ГМ/ГЭ/ГТ/метрологи/транспорт/ИТ/АХО в метаданных не выявлено — ожидается настройка маршрутов и ролевой адресации.

## 3. Подтвержденные факты
- Подтверждено структурой каталога и BSL: **BusinessProcesses/ЭкспертизаЗаявкиНаЗакупку**, **BusinessProcesses/КорректировкаЗаявкиНаЗакупку**.
- Подтверждено BSL (индекс методов): **ПервоначальноеЗаполнениеПараметровСогласованияБизнесПроцессовЗаявкиНаЗакупку** в общем модуле **Бизнес процессы заявок на закупку**.
- Подтверждено метаданными: механизм **Этапы универсальных процессов** (справочник) для настраиваемых маршрутов согласования в смежных подсистемах.
- Подтверждено BSL: модуль **Управление процессами согласования (переопределяемый)** содержит контуры условных переходов для заявок (связано с согласованием заявок на оплату — как базовый паттерн условных переходов).

## 4. Открытые вопросы / пробелы
- Не подтверждено в данном проходе: готовые маршруты с перечисленными службами «из коробки».
- Отчёт **Состояние потребности** ([строка 52]) — отдельная проверка отображения этапов.

## 5. Новые объекты системы
- При необходимости: предопределённые маршруты и роли под каждую службу.

## 6. Существующие объекты, требующие доработки
- Бизнес-процессы заявок на закупку, настройки этапов, связь с **Оперативный план** / **СПЗ**.

## 7. Зависимости

## 8. Ролевая модель и доступ
