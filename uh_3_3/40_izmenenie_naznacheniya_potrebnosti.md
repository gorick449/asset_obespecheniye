# Оценка: [строка 40] Изменение назначения потребности

## 1. Карточка оценки
- **Номер строки:** 40  
- **Название:** Изменение назначения потребности (переназначение, высвобождение)  
- **Краткая формулировка требования:** Операции переноса потребности и высвобождения резерва с отражением в запасах и отчётах по потребности.  
- **Дата оценки:** 23.03.2026  
- **Версия оценки:** 1.0  
- **Цель бизнеса:** Гибко перераспределять обеспечение без ручных обходных путей.  
- **Границы:** *Scope* — объекты назначения/обеспечения, команды переноса/снятия резерва, отчёты. *Out* — полная перестройка MRP вне текущей архитектуры.

## 2. Статус реализации
- **Полностью реализовано:** нет  
- **Частично реализовано:** да  
- **Не реализовано:** нет  
- **Не определено (требует проверки):** да — покрытие именно «назначения потребности» в смысле закупочного резерва  
- **Вероятно покрыто частично:** да  
- **Требует проверки:** единый UX для переназначения в документе обеспечения  
- **Пояснение:** В BSL-индексе найдена процедура «ПеренестиПотребность» в контексте обработки/формы «ПодборПотребностейИзПлана»; присутствуют модули «ОбеспечениеПотребностей», «ПланированиеПотребностей». Отдельный тип «НазначениеПотребностейЕХ» в дереве конфигурации соответствует определяемому типу (файл `DefinedTypes`), прямой объектной модели с реквизитами через выбранную операцию структуры не раскрыт. Процедура «ЗаполнитьВысвобождениеРезервов» в документе «КорректировкаПланов» относится к бюджетным лимитам (ТЧ «Требуется»/«ОбеспечитьЗаСчет»), что **не эквивалентно** без доп. проверки закупочному «высвобождению резерва ТМЦ».

## 3. Подтверждённые факты
- Наличие процедуры «ПеренестиПотребность» (объект «ПодборПотребностейИзПлана») — подтверждено BSL-индексом.  
- Наличие определяемого типа «НазначениеПотребностейЕХ» — подтверждено файловой структурой конфигурации.  
- Объекты с именами «ОбеспечениеПотребностей», «ПланированиеПотребностей» участвуют в коде планирования — подтверждено BSL-индексом.
