# Оценка: интеграция с КИС КССС (коды централизованных НСИ)

## 1. Карточка оценки
- **Номер строки:** 63
- **Название:** Интеграция с КИС КССС (коды централизованных НСИ)
- **Краткая формулировка требования:** Получение и учёт внешних (централизованных) кодов НСИ, согласованных с КИС КССС, для справочников и обменов.
- **Дата оценки:** 23.03.2026
- **Версия оценки:** 1.0
- **Цель бизнеса:** Единые идентификаторы НСИ в холдинге и корректное сопоставление при интеграциях и отчётности.
- **Границы доработки (scope / out-of-scope):**
  - **Scope:** определение перечня справочников; хранение внешнего кода; обработки загрузки/обновления; контроль обязательности; влияние на отчёты и обмены (требование 60).
  - **Out-of-scope:** разработка самой КИС КССС; изменение регламентов ЦНСИ на стороне владельца КИС без ТЗ.

## 2. Статус реализации
- **Полностью реализовано:** нет (как интеграция именно с КИС КССС)
- **Частично реализовано:** да
- **Не реализовано:** специфичный коннектор/маппинг под КИС КССС без уточнения протокола
- **Не определено:** транспорт и модель данных КИС КССС в вашем контуре
- **Вероятно покрыто частично:** механизмы управления НСИ и обмена с внешними ИБ
- **Требует проверки:** какие именно справочники должны получать код КССС
- **Пояснение статуса:** В конфигурации есть подсистема интеграции и управления НСИ (**ВнешниеИнформационныеБазы**, **СправочникиБД**, **ЗаявкаНаИзменениеНСИ**, отчёт **СинхронизацияНСИ**, обработки синхронизации и настройки правил). Объектов с именем «КИС КССС» в метаданных не найдено — привязка к конкретной КИС требует проектного слоя.

## 3. Подтверждённые факты
- Подтверждено метаданными: справочник **ВнешниеИнформационныеБазы** и **СправочникиБД** с назначением настройки справочников для интеграции с внешними ИБ и согласования (справка объекта).
- Подтверждено метаданными: документ **ЗаявкаНаИзменениеНСИ** и общие реквизиты с префиксом **НСИ_** (в глобальном поиске реквизитов встречается в т.ч. **НСИ_ВИБ** и др.) — контур управляемой НСИ.
- Подтверждено метаданными: отчёт **СинхронизацияНСИ**, обработки **СинхронизацияЭлементов**, **НастройкаПравилКонтроляНСИ** — инструменты сопоставления и правил обмена.
- Подтверждено поиском по метаданным: объектов с наименованием, содержащим «КССС», не обнаружено.

## 4. Открытые вопросы / пробелы
- Полный перечень справочников и реквизитов для кода КССС (номенклатура, контрагенты, ОКПД2, прочее).
- Периодичность обновления и политика при конфликте кода.
- Нужна ли блокировка операций при отсутствии кода.

## 5. Новые объекты системы
- При необходимости: регистр сведений «Коды КССС» или расширение существующих регистров соответствия — по проекту.
- Обработка/РЗ загрузки из КИС — если нет готового транспорта в текущей настройке обмена.

## 6. Существующие объекты, требующие доработки
- Целевые справочники (уточняются заказчиком): реквизиты внешнего кода, формы, проведение проверок заполнения.
- Модули правил синхронизации НСИ (**НастройкаПравилКонтроляНСИ** / связанные объекты).

## 7. Зависимости
- **Внутренние:** требование 10 (классификация ТРУ) для согласованности ключей; требование 60 (общая интеграционная политика).
- **Внешние:** доступ к КИС КССС, регламент выдачи кодов.

## 8. Ролевая модель и доступ
- Ограничить загрузку кодов ролями администратора НСИ / интеграции.
- Права на изменение контролируемых справочников — по существующему контуру **ЗаявкаНаИзменениеНСИ**.

## 9. Варианты реализации
1. **Минимально инвазивный:** добавить реквизиты/РС соответствия; регламентная загрузка из файла/API по спецификации КИС; проверки в формах ключевых справочников.
2. **Архитектурный:** выделенный адаптер КИС, единый регистр соответствий для всех обменов, централизованный мониторинг расхождений и отчётность.

## 10. Класс сложности

### Вариант «Минимально инвазивный»
- **Класс:** M
- **Обоснование:** ограниченный набор справочников и понятный поток обновления при наличии спецификации; умеренный риск регрессии НСИ.
- **Степень модификации:** средняя.

### Вариант «Архитектурный»
- **Класс:** L
- **Обоснование:** несколько объектов, единый контур идентификации, отчёты контроля, тесная связь с обменами.
- **Степень модификации:** высокая.

## 13. Гипотезы и варианты
- **Гипотезы:** КИС отдаёт коды пакетами; допускается файловый обмен на первом этапе.
- **Вероятность доработок:** средняя после получения модели данных КИС.
- **Проверить:** перечень справочников и обязательность кода по видам операций.

## 14. Антипаттерны и вклад в оценку
Подтвержденные антипаттерны в текущей реализации не выявлены.

## 15. Допущения и ограничения
- Допущение: не требуется миграция истории за период более одного отчётного года без отдельного ТЗ.
- Ограничение: при большом числе справочников (>10) оценка смещается к верхней границе диапазона.

## 16. Решение по оценке
- **Вывод:** требуется доработка; типовой контур НСИ есть, интеграция с КИС КССС как именованная — не подтверждена.
- **Рекомендованный диапазон:**
  - Минимально инвазивный: **5.0 / 8.5 / 14.0** чел.-дней
  - Архитектурный: **10.0 / 17.5 / 27.0** чел.-дней
- **Условия валидности:** фиксированный перечень справочников; один канал поставки кодов; без RLS-сложностей сверх типовых.

## 17. Проверка quality gates
- Gate 1: требуется уточнение перечня НСИ.
- Gate 2: подтверждено метаданными — инфраструктура НСИ/ВИБ есть, КИС КССС — нет.
- Gate 3: риски сопоставления зафиксированы.
- Gate 4: классы M и L с диапазонами.
- Gate 5: при расширении перечня справочников — пересчёт.
