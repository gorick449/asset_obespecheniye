# Оценка: [строка 46] Признак прослеживаемости, коды ТН ВЭД

## 1. Карточка оценки
- **Номер строки:** 46
- **Название:** Признак прослеживаемости и коды ТН ВЭД
- **Краткая формулировка:** классификация номенклатуры для прослеживаемости и указание ТН ВЭД с влиянием на поступления и РНПТ.
- **Дата оценки:** 23.03.2025
- **Версия оценки:** 1.0
- **Цель бизнеса:** корректный учёт прослеживаемых товаров и связь с РНПТ (47–50).
- **Границы:** реквизиты номенклатуры и типовые механизмы прослеживаемости; независимая классификация ТРУ (требование 10) — отдельно.

## 2. Статус реализации
- **Полностью реализовано:** **да** на уровне типовой подсистемы: в **Номенклатуре** присутствуют реквизиты **ПрослеживаемыйТовар**, **КодТНВЭД**, табличная часть **ИсторияПрослеживаемогоТовара** — подтверждено метаданными структуры объекта.
- **Частично:** влияние на конкретные сценарии поступления (импорт/ЕАЭС) требует настройки ФО **ВестиУчетПрослеживаемыхТоваров** и методики — подтверждено наличием константы/ФО в метаданных (см. семантический поиск и перечень объектов «Прослеживаемость»).

## 3. Подтверждённые факты
- **Справочники.Номенклатура:** реквизиты **ПрослеживаемыйТовар**, **КодТНВЭД**, **ПрослеживаемыйКомплект**, ТЧ **ИсторияПрослеживаемогоТовара** — подтверждено `object_structure`.
- Подсистема и объекты **Прослеживаемость товаров**, **Регистрация прослеживаемых товаров**, **Прослеживаемые товары** (регистр накопления) — подтверждено поиском по имени/описанию.

- Поведение проведения **Поступление товаров услуг** для строк с РНПТ — см. требования 47–49 (отдельно подтверждено наличием проверок РНПТ в модуле объекта поступления по BSL-индексу).
