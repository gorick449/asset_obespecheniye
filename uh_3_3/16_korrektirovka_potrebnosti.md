# Оценка покрытия: [строка 16] Корректировка потребности

## 1. Карточка оценки
- Номер строки: 16
- Название: Документ «Корректировка потребности»
- Краткая формулировка требования: корректировка количества/состава потребности с сохранением связи с исходной заявкой и влиянием на план потребности.
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: управляемые изменения без правки проведённых первичных документов.
- Границы: УХ 3.3, подсистема планирования потребностей.

## 2. Статус реализации
- Полностью реализовано: нет (по наименованию ТЗ)
- Частично реализовано: да
- Пояснение: в конфигурации присутствует документ **Замена номенклатуры** с синонимом **Корректировка потребностей**; также перечисление **Вид операции корректировки потребностей** и **Статус корректировки потребности**. Связь с планом потребностей подтверждается наличием типовых процедур планирования в менеджере документа.

## 3. Подтвержденные факты
- Подтверждено метаданными: документ **Замена номенклатуры** (синоним «Корректировка потребностей»); реквизиты периода закупки, ЦФО, организации; ТЧ **Номенклатура для замены** с колонками исходной и аналоговой номенклатуры, количеств, цен, признака **Отмена потребности**.
- Подтверждено BSL-кодом: в менеджере документа **Замена номенклатуры** присутствует экспортная функция подготовки параметров планирования потребностей (контур РН **Планирование потребностей**).
- Подтверждено BSL-кодом: модуль **Планирование потребностей** вызывается при заполнении по регистру в связке с корректировками (в т.ч. из процессов экспертизы заявки на обеспечение).

## 4. Открытые вопросы / пробелы
