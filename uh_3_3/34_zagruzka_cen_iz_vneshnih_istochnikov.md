# Оценка: [строка 34] Загрузка цен из внешних источников

## 1. Карточка оценки
- **Номер строки:** 34  
- **Название:** Ценообразование — загрузка цен из внешних источников  
- **Краткая формулировка требования:** Обеспечить получение цен из внешних источников (файл, обмен, при необходимости регламент/API), сопоставление с номенклатурой и использование в плановых ориентирах и сравнительном анализе.  
- **Дата оценки:** 23.03.2026  
- **Версия оценки:** 1.0  
- **Цель бизнеса:** Актуализировать справочные и закупочные цены без ручного ввода, снизить разрыв между рынком/прайсами поставщиков и внутренними ориентирами.  
- **Границы доработки (scope / out-of-scope):**  
  - *Scope:* доработка или настройка каналов загрузки, сопоставления номенклатуры, записи в регистры/документы цен, протокол ошибок, связь с отчётом сравнения цен при необходимости.  
  - *Out-of-scope:* юридическая валидация внешних данных, настройка конкретного контрагента-источника вне ИБ.

## 2. Статус реализации
- **Полностью реализовано:** нет  
- **Частично реализовано:** да  
- **Не реализовано:** выделенный контур «внешние цены конкурентов/рынка по API под требования заказчика» без проверки ТЗ  
- **Не определено (требует проверки):** наличие готового HTTP-сервиса именно для прайсов (не обмена НСИ в целом)  
- **Вероятно покрыто частично:** регламент «Загрузка данных внешних систем», универсальные обмены  
- **Требует проверки:** влияние загруженных цен на конкретный отчёт [строка 58] в требуемых разрезах  
- **Пояснение статуса:** Подтверждены типовые механизмы загрузки из файлов и обмена; прайс поставщика и виды цен поставщиков присутствуют в модели. Полное покрытие формулировки «внешние источники» в смысле произвольного API и единого протокола ошибок по ТЗ не подтверждено без уточнения.

## 3. Подтверждённые факты
- Обработка «Загрузка данных из внешних файлов»: в справке указано сопоставление позиций по штрихкоду, артикулу или наименованию и возможность уточнения сведений о ценах (и др.) после сопоставления — подтверждено метаданными (справка объекта).  
- Справочник «Виды цен поставщиков»: в справке указана связь с регистрацией цен документом «Регистрация цен поставщиков» — подтверждено метаданными.  
- Присутствуют регламентное задание «Загрузка данных внешних систем», обработки универсального обмена (в т.ч. XML/EnterpriseData), обмен НСИ — подтверждено перечнем объектов конфигурации.  
- Перечисление «Способы заполнения цен» и константа «Настройка заполнения цены покупки» — подтверждено метаданными (участвуют в контуре цен закупки).

## 4. Открытые вопросы / пробелы
- Требуется ли только файл или обязателен канал API; формат и частота.  
- Целевой регистр/тип цен: закупочные плановые, цены поставщика, отдельный «внешний ориентир».  
- Нужны ли дедупликация, версионность периода действия и обязательный протокол с повторной отправкой.

## 5. Новые объекты системы
- При варианте «отдельный API-контур»: регистр сведений протокола загрузки, опционально справочник профилей подключения, HTTP-сервис или внешняя обработка — по согласованию.

## 6. Существующие объекты, требующие доработки
- Обработки загрузки из файлов / универсального обмена; возможно документы установки/регистрации цен; отчёт сравнительного анализа цен при необходимости новых полей.

## 7. Зависимости
- **Внутренние:** номенклатура (идентификаторы сопоставления), типы цен, документы поступления/регистрации цен, подсистема закупок.  
- **Внешние:** внешняя система или файловый поставщик данных.

## 8. Ролевая модель и доступ
- Роли на обработки обмена и изменение цен; при новом регистре — чтение/изменение для службы закупок и администраторов интеграции.

## 9. Варианты реализации
1. **Минимально инвазивный:** донастройка существующих загрузок из файла и типов обмена, запись в имеющиеся объекты цен поставщика/номенклатуры, простой протокол в журнале регистрации.  
2. **Архитектурный:** выделенный сервис приёма цен, РС протокола и статусов строк, маппинг внешних кодов, регламентная обработка очереди.

## 10. Класс сложности
- **Вариант 1 (минимально инвазивный):** **M** — несколько объектов, сопоставление и правила записи; обоснование: используется существующая база загрузок и цен. Степень модификации: **средняя**.  
- **Вариант 2 (архитектурный):** **L** — новый интеграционный контур, протокол, возможны доработки отчётов; степень модификации: **высокая**.
