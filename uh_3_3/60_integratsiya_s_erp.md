# Оценка: [строка 60] Интеграция с ERP

## 1. Карточка оценки
- Номер строки: 60
- Название: Интеграция с ERP
- Краткая формулировка требования: обмен с учётной ERP заказчика (конкретные объекты и направления уточняются); регламенты и идентификаторы сопоставления; отсутствие дублирования с прочими интеграциями [строки 62–64].
- Дата оценки: 23.03.2026
- Версия оценки: 1
- Цель бизнеса: согласованность мастер-данных и операционных фактов между 1С:УХ и ERP.
- Границы доработки (scope / out-of-scope): проектирование и реализация обмена в рамках согласованного перечня объектов; не «вся ERP целиком» без ТЗ.

## 2. Статус реализации
- Полностью реализовано: не подтверждено (ERP заказчика не идентифицирована).
- Частично реализовано: в конфигурации есть **общая инфраструктура обмена**: подсистемы и модули интеграции с внешними системами, **ОбменНСИ**, **ВнешниеИнформационныеБазы**, пакеты **EnterpriseDataExchange**, веб-сервис **EnterpriseDataExchange_1_0_1_1**, подсистема **СкопированоИзERP** в блоке централизованного управления закупками — подтверждено анализом метаданных.
- Не реализовано: специфическая связка с конкретной ERP заказчика без отдельного проекта — не подтверждается.
- Требует проверки: пересечение с интеграциями [строки 61–64] и с поступлением/заказами [строки 37–38].
- Пояснение статуса (1-3 предложения): Технологические кирпичи для обмена присутствуют; содержание «интеграции с ERP» полностью определяется внешним контуром заказчика, поэтому оценка носит диапазонный характер с высоким резервом.

## 3. Подтверждённые факты
- Обработка **ОбменНСИ**, справочник **ВнешниеИнформационныеБазы**, обработка **НастройкаПравилЗаполненияОбъектовБД** и отчёт **СинхронизацияНСИ** присутствуют — подтверждено анализом метаданных.
- Подсистема **СкопированоИзERP** существует — подтверждено анализом метаданных.
- Пакет XDTO и веб-сервис **EnterpriseDataExchange_1_0_1_1** существуют — подтверждено анализом метаданных.

## 9. Варианты реализации — если релевантно и есть неопределённость
- **Вариант 1:** типовой/полутиповой обмен по **Enterprise Data Exchange** или правилам НСИ для ограниченного набора объектов.
- **Вариант 2:** кастомный коннектор (HTTP + XDTO/JSON) с очередью и протоколом ошибок для нетиповой ERP.

## 10. Класс сложности

- **Вариант 2:** **XL** как базовый класс для нетиповой ERP.
