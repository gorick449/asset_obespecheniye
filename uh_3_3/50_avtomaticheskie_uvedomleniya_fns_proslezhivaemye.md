# Оценка: [строка 50] Автоматическое формирование уведомлений об операциях с прослеживаемыми товарами

## 1. Карточка оценки
- **Номер строки:** 50
- **Название:** Уведомления ФНС по прослеживаемым товарам
- **Краткая формулировка:** автоматизация подготовки и отправки уведомлений (остатки, ввоз, перемещения, отчёт об операциях) для прослеживаемости.
- **Дата оценки:** 23.03.2025
- **Версия оценки:** 1.0
- **Цель бизнеса:** соблюдение требований ФНС по прослеживаемости с минимизацией ручного труда.
- **Границы:** типовой контур **1С** (документы уведомлений, помощник, регламентированная отчётность); не путать с **e-mail рассылками пользователям** (требование 41).

## 2. Статус реализации
- **Частично реализовано / полуавтомат:** в конфигурации есть **документы уведомлений** (**Уведомление об остатках прослеживаемых товаров**, **Уведомление о ввозе прослеживаемых товаров**, **Уведомление о перемещении прослеживаемых товаров**), **обработка Помощник получения РНПТ** с пошаговым сценарием (инвентаризация → создание уведомлений → отправка в ФНС через **1С-Отчетность** → ожидание ответа → запись РНПТ), **журнал ПрослеживаемостьУведомления**, регламентированные отчёты (в т.ч. **Отчёт об операциях с товарами, подлежащими прослеживаемости**) — подтверждено справками объектов и семантическим поиском.
- **Полностью автоматическое** формирование и отправка **без участия пользователя** для всех видов операций **не утверждается** по результатам метаданных (в справках указаны ручные действия и кнопки, альтернатива — выгрузка в файл).

## 3. Подтверждённые факты
- **Помощник получения РНПТ** описывает автоматические проверки остатков и **предложение создать уведомления**, отправку через **1С-Отчетность** или **выгрузку в файл** — подтверждено справкой обработки.
- Документ **Уведомление о ввозе** может создаваться из **Поступления** по гиперссылке и через помощник — подтверждено справкой документа.
- Модуль **ИнтерфейсыВзаимодействияБРОВызовСервера** содержит функцию поиска **квитанций о присвоении РНПТ** по параметрам уведомления — подтверждено BSL-карточкой рутины (связь с контуром БРО/ФНС).

## 4. Пробелы
- Покрытие **каждого** хозяйственного события автоматическим созданием уведомления «в фоне» без сценарного анализа всех документов — **не подтверждено**.
- Требования к **регламентным заданиям** именно на автосоздание уведомлений — уточнить (в оценке не найдено отдельного задания с таким именем в выборке; есть задание **ПроверкаРНПТ**, что относится к 47).

## 5. Новые объекты
- При требовании «полный автомат»:** регламентные задания, подписки на проведение, очередь ошибок — см. вариант 2.

## 6. Существующие объекты
- Документы уведомлений, **ПомощникПолученияРНПТ**, регламентированные отчёты прослеживаемости, **ПрослеживаемостьКлиент.ВыгрузитьУведомлениеПоПрослеживаемости** (по BSL-индексу).

## 7. Зависимости
- **Внешние:** **1С-Отчетность**, оператор, криптография.
- **Внутренние:** 46–49 (корректные остатки и РНПТ).

## 8. Роли
- Права на работу с уведомлениями и выгрузкой в ФНС.

## 9. Варианты реализации
1. **Типовой полуавтомат:** помощник + документы + регламентированные отчёты; обучение.
2. **Расширение автоматизации:** регламентные задания и обработчики проведения для автосоздания черновиков уведомлений и протоколирования.

## 10. Класс сложности

- **Вариант 2:** **L** (много типов операций и рисков)
