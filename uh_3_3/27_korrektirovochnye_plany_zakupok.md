# Оценка: [строка 27] Формирование корректировочных планов закупок

## 1. Карточка оценки
- Номер строки: 27
- Название: Формирование корректировочных планов закупок
- Краткая формулировка: внесение изменений в размещённый/утверждённый план с сохранением истории и повторным согласованием при необходимости.
- Дата оценки: 23.03.2026
- Версия: 1.0
- Цель бизнеса: легитимная корректировка плана без потери учётной истории и соответствия 223-ФЗ/внутренним регламентам.
- Границы: документы плана закупок, версии, обмен ЕИС; out-of-scope — юридическое сопровождение изменений.

## 2. Статус реализации
- Полностью реализовано: сценарий корректировки через **новые версии** документов «Строка плана закупок» и «Программа закупок» описан в справке к «Строка плана закупок» и «Программа закупок» (ввод на основании, версии, выгрузка на ЕИС).
- Частично реализовано: отдельный специализированный документ «Корректировка плана закупок» **не выявлен** как отдельное имя объекта в семантическом топе; используется модель версий.
- Пояснение: подтверждено **метаданными (справка)**; влияние на исполнение (блокировки) — уточнять BSL.

## 3. Подтверждённые факты
- Регистр «Строки плана закупок» и реквизиты версии/идентификатора строки поддерживают учёт нескольких версий.
- «Программа закупок» содержит номер версии и механизм новой версии для выгрузки изменений (описано в справке).

## 4. Пробелы
- Явные серверные запреты на закупку по старой версии после корректировки — проверить в коде проведения/регистров.

## 5. Новые объекты
- Обычно не требуются при принятии модели версий.

## 6. Существующие объекты
- «Строка плана закупок», «Программа закупок», регистр «Строки плана закупок», обработки обмена с ЕИС.

## 7. Зависимости
- 24, 26, 56.

## 8. Ролевая модель
- Права на создание новой версии и обмен; согласование как для исходного плана.

## 9. Варианты
- **А:** использовать типовые версии без доработок.
- **Б:** добавить прикладной «пакет корректировки» (отчёт/помощник) для массовых изменений.

## 10. Класс сложности
