# Оценка покрытия: [строка 22] Заявки на закупку и строки плана закупок

## 1. Карточка оценки
- Номер строки: 22
- Название: Формирование заявок на закупку, строки плана закупок
- Краткая формулировка требования: документ(ы) заявки, включение в план/программу закупок, связь с планом потребности и ID потребности, переход к заказу поставщику.
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: мост между потребностью и закупочной процедурой.
- Границы: УХ 3.3 корпоративные закупки.

## 2. Статус реализации
- Полностью реализовано: нет
- Частично реализовано: да
- Пояснение: отдельного документа с именем «Заявка на закупку» в списке документов выборки не найдено; в коде и загрузках используется профиль **Заявка на закупку** для **Оперативного плана**. **Строка плана закупок** (СПЗ) формируется из потребности (**Заявка на обеспечение**) и через **Помощник планирования**; **Программа закупок** агрегирует СПЗ. Связь с планом потребности подтверждается модулями **Планирование потребностей**. Объект **Идентификаторы строк планов закупок** (справочник/регистр в метаданных) поддерживает учёт идентификаторов.

## 3. Подтвержденные факты
- Подтверждено метаданными: документ **Строка плана закупок**; справка описывает подбор из плана потребности по **Заявке на обеспечение**, связь с **Программой закупок**, автоматическое формирование через **Помощник планирования**.
- Подтверждено метаданными: документ **Программа закупок** с табличными частями ссылок на **Строка плана закупок**.
- Подтверждено BSL: **ПолучитьПланыПоЗаявкеНаЗакупку** в модуле **Планирование потребностей**; обработки загрузки с префиксом **ЗаявкаНаЗакупку**.
- Подтверждено метаданными: объект **Идентификаторы строк планов закупок** (структура с полями периода, организации, владельца и др. по выгрузке метаданных).
