# Оценка: [строка 54] Отчёт «Состояние запасов»

## 1. Карточка оценки
- Номер строки: 54
- Название: Состояние запасов (резервы под потребности, невостребованное, аварийные, страховые, ДХНО)
- Краткая формулировка требования: классификация запасов по категориям согласно нормативам и назначениям; корректность резервов под потребности.
- Дата оценки: 23.03.2026
- Версия оценки: 1
- Цель бизнеса: управленческий контроль структуры запасов и соответствия нормам [строка 11] и назначениям [строка 40].
- Границы доработки (scope / out-of-scope): отчёт и при необходимости донастройка нормативов/резервирования; не пересмотр всей политики складского учёта.

## 2. Статус реализации
- Полностью реализовано: не подтверждено (категории «ДХНО», «невостребованное» в явном виде в названиях отчётов не искались в полном перечне).
- Частично реализовано: вероятно — типовые отчёты по остаткам/движению товаров и механизм **ИспользоватьРезервирование** (константа/ФО) дают базу для резервов.
- Не реализовано: единый отчёт с полным набором категорий из формулировки — не подтверждено.
- Не определено (требует проверки): алгоритм отнесения к «аварийным/страховым» через нормативы [строка 11].
- Вероятно покрыто частично: **ДвижениеТоваров** и отчёты по запасам МСФО относятся к другой предметной области (обесценение, резервы по долгам), не к резервам под заказы.
- Требует проверки: регистры резервирования под потребность в подсистеме закупок/склада.
- Пояснение статуса (1-3 предложения): Семантический поиск по «состояние запасов, резерв» в категории «Отчеты» вернул в основном отчёты по **резервам** в бухгалтерском/кадровом смысле, а не складские резервы под потребность. Специализированный контур под формулировку требования требует отдельной привязки к регистрам резервирования и нормативам.

## 3. Подтверждённые факты
- **Константы.ИспользоватьРезервирование** и одноимённая функциональная опция существуют — подтверждено анализом метаданных.
- Отчёт **ДвижениеТоваров** существует (остатки и обороты по счетам запасов) — подтверждено анализом метаданных.
- Прямого попадания в выборку отчётов по запросу «состояние запасов + резерв под потребность» не зафиксировано — подтверждено результатом семантического поиска (как пробел подбора).

## 4. Открытые вопросы / пробелы (если есть)
- Определение ДХНО в терминах учётной политики заказчика.
- Источник «невостребованных» запасов (свободные остатки без заказов/назначений).

## 5. Новые объекты системы
- При необходимости: регистр сведений «Классификация запасов по политике» или расширение нормативов [строка 11] — только если нельзя вычислить из существующих регистров.

## 6. Существующие объекты, требующие доработки
- Отчёт(ы) по остаткам/обеспечению (после уточнения имён в подсистеме склада).
- Связка с **ИзменениеНазначения** / механизмами назначения [строка 40] — по результатам уточнения имён документов.

## 7. Зависимости
- Внутренние: нормативы [строка 11], назначения [строка 40], резервирование, складские остатки.
- Внешние: не подтверждены.

## 8. Ролевая модель и доступ

## 9. Варианты реализации — если релевантно и есть неопределённость
- **Вариант 1:** отчёт на СКД с расчётом категорий из остатков, резервов и нормативов.
- **Вариант 2:** предрасчёт категорий регламентным заданием в РС для производительности на больших объёмах.
