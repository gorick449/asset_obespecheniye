# Оценка: [строка 30] Ввод данных о победителе тендера

## 1. Карточка оценки
- Номер строки: 30
- Название: Ввод данных о победителе тендера
- Краткая формулировка: фиксация результата (победитель, цены/условия) с привязкой к лоту и контрагенту; загрузка из ИСУ Тендер или ручной ввод.
- Дата оценки: 23.03.2026
- Версия: 1.0
- Цель бизнеса: юридически значимый результат процедуры для договора и заказа.
- Границы: «Протокол выбора победителей», «Предложение поставщика», интеграционные модули ЦУЗ–ЭТП; out-of-scope — бизнес-решение на площадке.

## 2. Статус реализации
- Полностью реализовано: документ «Протокол выбора победителей» с оценкой предложений, переторжкой, формированием договора по победителю описан в справке метаданных; документ «Предложение поставщика» хранит условия по номенклатуре, цены, НДС, валюту.
- Частично реализовано: **прямая интеграция с ЭТП не преднастроена** (указано в справке к «Закупочные процедуры»); есть общий модуль «ИнтеграцияЦУЗсЭТПСерверУХ» с функциями получения протокола/описания предложений (подтверждено поиском кода по описанию).
- Пояснение: подтверждено **метаданными и BSL-документацией рутин** (не полный обход всех веток).

## 3. Подтверждённые факты
- «Протокол выбора победителей»: ветки решений (состоялась/переторжка/не состоялась/единственный поставщик), связь с лотами и графиком поставок в договоре — в справке.
- «Предложение поставщика»: ТЧ «Условия по номенклатуре» с ценой, ставкой НДС, количеством, валютой документа в шапке.
- Модуль «ИнтеграцияЦУЗсЭТПСерверУХ» содержит экспортные функции работы с протоколом и описанием предложений победителей.

## 4. Пробелы
- Полный сценарий «загрузка из ИСУ Тендер» для конкретной площадки — зависит от требования 61 и настроек.

## 5. Новые объекты

## 6. Существующие объекты
