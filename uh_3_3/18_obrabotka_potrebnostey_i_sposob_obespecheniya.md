# Оценка покрытия: [строка 18] Обработка потребностей, выбор способа обеспечения

## 1. Карточка оценки
- Номер строки: 18
- Название: Обработка потребностей, выбор способа обеспечения
- Краткая формулировка требования: рабочее место со списком потребностей, запись решения для последующих документов (заказ, перемещение, склад).
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: централизованное решение по закрытию потребностей.
- Границы: УХ 3.3; связь с ERP-контуром заказов/перемещений — на уровне наличия типовых механизмов.

## 2. Статус реализации
- Полностью реализовано: нет
- Частично реализовано: да
- Пояснение: присутствует документ **Обеспечение потребностей** с ТЧ **Потребности в номенклатуре** (количества, аналитики планирования, суммы); выбор способа детализируется в связке с перечислением способов (см. [строка 19]). Дополнительно в конфигурации есть обработки подбора потребностей (**Подбор потребностей из плана**, **Помощник планирования**).

## 3. Подтвержденные факты
- Подтверждено метаданными: документ **Обеспечение потребностей**; табличная часть **Потребности в номенклатуре** с аналитиками источника и планирования, количеством и суммами.
- Подтверждено метаданными: обработка **Подбор потребностей из плана** и **Помощник планирования** работают с остатками/потребностью для закупочного контура.
- Подтверждено BSL: модули документа **Обеспечение потребностей** присутствуют (детальная трассировка проведения в данном проходе не раскрывалась полностью).

## 4. Открытые вопросы / пробелы
- Требуется чтение проведения **Обеспечение потребностей** для подтверждения автоматического порождения **Заказа поставщику** / **Перемещения** по каждому способу.

## 5. Новые объекты системы
- Обычно не требуются при достаточности **Обеспечение потребностей**.

## 6. Существующие объекты, требующие доработки
- **Обеспечение потребностей** — расширение алгоритмов и связей.
- Общие модули планирования/закупок.

## 7. Зависимости
- Внутренние: РН **Планирование потребностей**, **Строка плана закупок**, заказы/склад (типовой контур).
- Внешние: настройки обеспечения в ERP-части конфигурации.

## 8. Ролевая модель и доступ
