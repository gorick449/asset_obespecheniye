# Оценка: [строка 47] Автоматическая проверка РНПТ через веб-сервис ФНС

## 1. Карточка оценки
- **Номер строки:** 47
- **Название:** Проверка РНПТ через веб-сервис ФНС
- **Краткая формулировка:** автоматизированная сверка РНПТ с данными ФНС, журналирование, настройки; вопрос обязательности перед проведением.
- **Дата оценки:** 23.03.2025
- **Версия оценки:** 1.0
- **Цель бизнеса:** снизить риск некорректных РНПТ в учёте и отчётности.
- **Границы:** штатный механизм конфигурации; доработки только при нестандартных политиках блокировки проведения.

## 2. Статус реализации
- **Полностью реализовано:** **да** как типовой контур: общий модуль **ПроверкаРНПТ**, **ПроверкаРНПТРегламентноеЗадание**, константа **ИспользоватьПроверкуРНПТ**, клиентские/серверные модули, форма включения **ВключениеПроверкиРНПТ**, отчёт **АнализРНПТДокументов** — подтверждено метаданными и BSL (поиск методов и описаний процедур).
- **Процессный контур (BSL):** процедуры **ПроверитьРНПТВДокументе**, **ПроверитьРНПТВДокументеФоновоеЗадание**, **ПолучитьСведенияОРНПТИзФНС**, **ЗаписатьСведенияОРНПТИзФНС** — подтверждено BSL-индексом и карточками рутин.

## 3. Подтверждённые факты
- Регламентное задание **ПроверкаРНПТ** вызывает **ПроверитьРНПТРегламентноеЗадание** — описание рутины указывает получение сведений из сервиса ФНС и проверку документов — подтверждено BSL.
- Результаты проверки сохраняются (процедура **СохранитьРезультатПроверкиРНПТ** присутствует в модуле) — подтверждено BSL-индексом.
- В модуле объекта **ПоступлениеТоваровУслуг** встречаются вызовы/логика статуса проверки РНПТ (по результатам точечного поиска **РНПТ** в ObjectModule) — подтверждено BSL.

## 4. Пробелы
- Явный запрет проведения при «красном» статусе для **всех** типов документов без чтения конкретных условий в модулях — **не подтверждён полностью** в данной оценке (требуется точечный разбор процедур проведения при необходимости жёсткого gate).

## 5. Новые объекты
- Не требуются при типовом использовании.

## 6. Существующие объекты
- **ПроверкаРНПТ**, **ПроверкаРНПТКлиент**, **НастройкиПроверкиРНПТ**, **СведенияОРНПТИзФНС** (регистр сведений по имени из описаний рутин), регламентное задание, отчёт анализа.

## 7. Зависимости
- **Внешние:** доступ к сервисам ФНС, параметры **1С-Отчетности**/интернет-соединение (типовые требования платформы).
- **Внутренние:** корректные РНПТ в документах (48–49).

## 8. Роли
- Права на использование проверки (**ЕстьПравоНаИспользованиеПроверки** в модуле) — подтверждено BSL.

## 9. Варианты реализации
1. **Типовая настройка** включения, расписание регламента, обучение.
2. **Ужесточение:** доработка проведения документов с обязательной успешной проверкой (если типовой механизм допускает обход).

## 10. Класс сложности

- **Вариант 2:** **S–M** в зависимости от числа типов документов
