# Оценка: [строка 59] Анализ ТРУ за 365 дней

## 1. Карточка оценки
- Номер строки: 59
- Название: Анализ ТРУ за 365 дней
- Краткая формулировка требования: движение номенклатуры в разрезе контрагента, договора, документа движения, счёта учёта; группировка по пунктам перечня ТРУ; связь с трассировкой потребности [строка 25].
- Дата оценки: 23.03.2026
- Версия оценки: 1
- Цель бизнеса: годовая аналитика закупок и потребления по корпоративной классификации ТРУ.
- Границы доработки (scope / out-of-scope): отчёт на базе учётных движений и НСИ [строка 10]; не построение внешнего хранилища.

## 2. Статус реализации
- Полностью реализовано: не подтверждено.
- Частично реализовано: вероятно — отчёт **ДвижениеТоваров** и прочие регламентированные отчёты по оборотам; они не ориентированы на формулировку «пункт перечня ТРУ».
- Не реализовано: отчёт с явным именем/назначением под «ТРУ 365» в метаданных — не найден (поиск по «ТРУ» в имени объекта дал нерелевантные совпадения из-за шума).
- Требует проверки: наличие реквизита связи номенклатуры с перечнем ТРУ [строка 10] и его использование в СКД.
- Пояснение статуса (1-3 предложения): Базовые движения по номенклатуре в конфигурации отражаются стандартными отчётами. Специализированная группировка по перечню ТРУ и сквозной ID потребности — высокая вероятность доработки СКД или нового отчёта.

## 3. Подтверждённые факты
- Отчёт **ДвижениеТоваров** существует и по справке формирует остатки и обороты по счетам запасов в количественном и суммовом выражении — подтверждено анализом метаданных (текст справки).
- Поиск объектов по подстроке имени «ТРУ» не выделил целевой отчёт анализа ТРУ — подтверждено анализом метаданных (как индикатор отсутствия прямого именования).

## 4. Открытые вопросы / пробелы (если есть)

## 5. Новые объекты системы
