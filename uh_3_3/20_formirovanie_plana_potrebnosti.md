# Оценка покрытия: [строка 20] Формирование плана потребности

## 1. Карточка оценки
- Номер строки: 20
- Название: Формирование плана потребности
- Краткая формулировка требования: консолидация заявок/корректировок с правилами отбора, фиксация в документе/регистре, разрезы для ПЗ.
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: утверждаемая база потребности для закупок.
- Границы: РН **Планирование потребностей** и документы-источники в УХ 3.3.

## 2. Статус реализации
- Полностью реализовано: нет
- Частично реализовано: да
- Пояснение: план потребности фиксируется в **РН Планирование потребностей** (ресурсы количество/суммы; измерения аналитик планирования и источника; реквизиты движений **Источник версии**, **Корректировка документа**). Источники включают **Заявка на обеспечение**, **Замена номенклатуры**, **Строка плана закупок** и др. (по составу модулей с процедурой подготовки параметров планирования). Отдельного документа «План потребности» как единственного носителя в выборке имён не выявлено — консолидация регистровая.

## 3. Подтвержденные факты
- Подтверждено метаданными: структура РН **Планирование потребностей** (измерения, ресурсы, реквизиты движения).
- Подтверждено BSL/метаданными: несколько документов содержат **ПодготовитьПараметрыПланированияПотребностей** (в т.ч. **Заявка на обеспечение**, **Замена номенклатуры**, **Строка плана закупок**, **Лот**, **Протокол выбора победителей**).
- Подтверждено метаданными и справкой: **Помощник планирования** формирует строки плана закупок из потребности по правилам **Положение о закупках**.

## 4. Открытые вопросы / пробелы
- Юридическое оформление «плана потребности» как отдельного документа vs регистр — уточнить у заказчика.

## 5. Новые объекты системы
- Опционально: документ-оболочка «План потребности» с движениями в тот же РН.
