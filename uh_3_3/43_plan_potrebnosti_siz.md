# Оценка: [строка 43] Планирование потребности в СИЗ

## 1. Карточка оценки
- **Номер строки:** 43  
- **Название:** Планирование потребности в СИЗ  
- **Краткая формулировка требования:** План закупки в разрезе номенклатуры и характеристик по статистике потребления, нормам и резервам; связь с общим планом закупок (строка 24).  
- **Дата оценки:** 23.03.2026  
- **Версия оценки:** 1.0  
- **Цель бизнеса:** Согласовать закуп СИЗ с фактической потребностью и нормами.  
- **Границы:** *Scope* — обработка/документ плана, алгоритмы трёх источников, выгрузка в ПЗ. *Out* — производственное планирование сырья.

## 2. Статус реализации
- **Полностью реализовано:** нет  
- **Частично реализовано:** да  
- **Не реализовано:** нет  
- **Не определено (требует проверки):** да — наличие **отдельного** «плана СИЗ»  
- **Вероятно покрыто частично:** да — через общее планирование потребностей/закупок  
- **Требует проверки:** сведение статистики выдачи и норм (строка 42)  
- **Пояснение:** Общие объекты планирования закупок и потребностей в конфигурации присутствуют (см. оценки по строкам 20–24); специализированного объекта «план СИЗ» в быстром поиске по смыслу **не подтверждено**. Подсистема спецодежды даёт учётные документы и отчёты по выданным материалам (например, отчёт «Материалы, выданные сотрудникам» — упомянут в метаданных). Итог: требование **вероятно** реализуемо как **надстройка** над общим планом + нормами, но не как готовый монолитный объект.

## 3. Подтверждённые факты
- Наличие подсистемы и опций учёта спецодежды — подтверждено метаданными.  
- Наличие отчёта по выданным материалам сотрудникам (учёт в эксплуатации) — подтверждено метаданными.  
- Общий контур «СтрокиПланаЗакупок»/планирования закупок используется в смежных оценках — логическая связь с строкой 24.

## 4. Открытые вопросы / пробелы
- Периодичность плана (месяц/квартал) и горизонт резерва.  
- Источник статистики: движения выдачи, списания, перемещения между сотрудниками.  
- Нужна ли отдельная категория ТРУ для СИЗ в плане.

## 5. Новые объекты системы

## 6. Существующие объекты, требующие доработки
