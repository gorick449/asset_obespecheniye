# Оценка покрытия: [строка 21] Согласование плана потребности

## 1. Карточка оценки
- Номер строки: 21
- Название: Согласование плана потребности
- Краткая формулировка требования: маршрут, статус «утверждён», блокировка заявок на закупку/выгрузок до утверждения; оповещения ([строка 41]).
- Дата оценки: 2026-03-23
- Версия оценки: 1.0
- Цель бизнеса: контрольная точка перед закупочным контуром.
- Границы: **Оперативный план** и связанные процессы; оповещения — отдельное требование.

## 2. Статус реализации
- Полностью реализовано: нет
- Частично реализовано: да
- Пояснение: **Оперативный план** поддерживает универсальные процессы согласования и опцию формирования движений лимитов только в статусе «Утверждён» (описано в справке документа). Блокировка **Строки плана закупок** / выгрузок до утверждения **программы закупок** требует сквозной проверки с [строка 22] и [строка 24] — в данном проходе не подтверждена полностью кодом отбора.

## 3. Подтвержденные факты
- Подтверждено текстом справки метаданных **Оперативный план**: возможность согласования по универсальным маршрутам; константа о формировании движений установки лимитов только в статусе «Утверждён».
- Подтверждено метаданными: реквизит **Вне маршрута** для обхода согласования суперпользователем.
- Подтверждено метаданными: документ **Программа закупок** с **Дата утверждения** — точка утверждения закупочного плана (смежный контур).

## 4. Открытые вопросы / пробелы
- Явная блокировка создания **Строки плана закупок** при неутверждённом плане потребности — проверить в модулях документа и обработках.
- Оповещения ([строка 41]) — не входят в глубину данного отчёта.

## 5. Новые объекты системы
