# Оценка: [строка 61] Интеграция с ИСУ Тендер

## 1. Карточка оценки
- Номер строки: 61
- Название: Интеграция с ИСУ Тендер (лоты, победитель, цены)
- Краткая формулировка требования: исходящая передача лотов и входящая загрузка результатов; сопоставление ключей лота/позиции; связь с загрузкой цен победителя [строка 33].
- Дата оценки: 23.03.2026
- Версия оценки: 1
- Цель бизнеса: исключить ручной перенос тендерных итогов и цен между ИСУ Тендер и 1С.
- Границы доработки (scope / out-of-scope): обмен по согласованному API/формату ИСУ; не разработка самой ИСУ.

## 2. Статус реализации
- Полностью реализовано: не подтверждено для ИСУ Тендер как внешней системы заказчика.
- Частично реализовано: в конфигурации есть **УправлениеИнтеграциейЦУЗсЭТП** (обмен с ЕИС/ЭТП), модули **ИнтеграцияЦУЗсЭТП*** , **ИнтеграцияСЕИС**, обработки **ИнтеграцияСЕИС_v40** с логикой лотов и предложений в XDTO — подтверждено анализом метаданных и BSL; это близко по смыслу (лоты, предложения), но не тождественно бренду «ИСУ Тендер».
- Не реализовано: готовый коннектор «ИСУ Тендер» по имени — не подтверждён.
- Требует проверки: совместимость форматов ИСУ с существующими XDTO ЕИС/ЭТП.
- Пояснение статуса (1-3 предложения): Технологический задел по работе с лотами и предложениями в закупочном контуре есть. Подключение именно ИСУ Тендер — проект согласования протокола и маппинга ключей.

## 3. Подтверждённые факты
- Обработка **УправлениеИнтеграциейЦУЗсЭТП** существует; в справке указано назначение для выгрузки на электронные площадки и ЕИС — подтверждено анализом метаданных.
- В формах **ИнтеграцияСЕИС_v40** присутствуют серверные функции заполнения структур **purchaseProtocol**, **lotApplications** (предложения по лотам) — подтверждено BSL (поиск по описанию).
- Документ **ПротоколВыбораПобедителей** и связанная логика победителя используются в закупках — подтверждено BSL.

## 4. Открытые вопросы / пробелы (если есть)
