# Оценка: [строка 44] Учёт выдачи СИЗ и карточки учёта СИЗ

## 1. Карточка оценки
- **Номер строки:** 44
- **Название:** Учёт выдачи СИЗ, формирование карточек учёта СИЗ
- **Краткая формулировка требования:** фиксировать выдачу СИЗ сотрудникам, иметь карточку учёта, обеспечить данные для отчёта обеспеченности (53).
- **Дата оценки:** 23.03.2025
- **Версия оценки:** 1.0
- **Цель бизнеса:** соблюдение требований по учёту СИЗ и прозрачность выдачи для планирования и отчётности.
- **Границы доработки (scope / out-of-scope):**  
  **Scope:** сопоставление с типовым контуром «спецодежда и инвентарь», печатные формы, отчётность, связь с номенклатурой СИЗ.  
  **Out-of-scope:** ведение нормативов (42) и плана (43) как отдельных задач; полная кадровая подсистема.

## 2. Статус реализации
- **Полностью реализовано:** нет (зависит от толкования «карточки учёта СИЗ»).
- **Частично реализовано:** да — типовой учёт выдачи спецодежды/инвентаря и признак СИЗ в номенклатуре.
- **Не реализовано:** отдельный документ с наименованием «выдача СИЗ», отдельная печатная форма строго по форме МБ-4 (если она зафиксирована в НПА заказчика) — не подтверждены метаданными.
- **Не определено (требует проверки):** достаточность печатной формы «МБ-7» вместо требуемой заказчиком «карточки».
- **Вероятно покрыто частично:** отчёт «Обеспеченность СИЗ» (53) при условии настройки номенклатуры, норм и полноты движений.
- **Требует проверки:** юридическое соответствие перечня печатных форм ожиданиям охраны труда.
- **Пояснение статуса:** в конфигурации есть контур передачи материалов в эксплуатацию со специализированными закладками и отчётом по выданным сотрудникам; прямой одноимённой «карточки учёта СИЗ» в найденных макетах документа передачи не выявлено (есть МБ-7).

## 3. Подтверждённые факты
- В справочнике **Номенклатура** есть реквизит **СредствоИндивидуальнойЗащиты** (тип **Булево**) — подтверждено метаданными.
- Включение учёта спецодежды/спецоснастки задаётся функциональной опцией и константой **ВедетсяУчетСпецодеждыСпецоснастки** — подтверждено метаданными.
- Документ **Передача материалов в эксплуатацию** по справке предназначен для передачи в эксплуатацию спецодежды, спецоснастки, инвентаря; на закладках указывается **физическое лицо**; печатные формы включают **ведомость МБ-7**, **накладная М-11** — подтверждено справкой объекта.
- У документа в метаданных зарегистрирован макет **ПФ_MXL_МБ7** — подтверждено метаданными.
- Отчёт **Материалы, выданные сотрудникам** формируется по данным **Передача материалов в эксплуатацию** (и расхода материалов) и забалансовых счетов МЦ.02/МЦ.04 — подтверждено справкой отчёта.
- Справочник **Назначения использования номенклатуры** содержит **количество по нормативу выдачи** для подстановки в документы — подтверждено справкой объекта.

## 4. Открытые вопросы / пробелы
- Нужна ли заказчику именно форма **МБ-4** (или иной регламентированный бланк «карточки учёта»), или достаточно **МБ-7/М-11**?
- Требуется ли отдельный склад/группа «СИЗ» или достаточно аналитики по номенклатуре и счетам учёта (см. требование 45)?
- Какие обязательные реквизиты для отчёта 53 (размеры, характеристики) должны попадать в документ выдачи — подтвердить по методике отчёта 53.

## 5. Новые объекты системы

- При варианте «минимально инвазивный»:** новых объектов может не потребоваться.

## 6. Существующие объекты, требующие доработки
- **ПередачаМатериаловВЭксплуатацию** — команды печати, при необходимости дополнительные реквизиты/ТЧ под отчёт 53.

- **МатериалыВыданныеСотрудникам** / связанные запросы — при необходимости новые поля для отчёта 53.
